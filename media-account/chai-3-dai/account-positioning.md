# 账号定位方案：「拆³代」

> **核心理念：三层递进，不绕弯子，只拆真相。**

## 一、一句话定位

**拆³代** = 拆信息 + 拆框架 + 拆惯性。

一个帮你在信息洪流中看清本质、理清结构、照见自己的思考类内容品牌。

## 二、为什么是“³”？

三层不是并列关系，是**一层比一层深，一层比一层值钱**的递进关系：

```
第一层：拆信息  →  你被信息淹没了 → 帮你浮出水面
                    ↓
第二层：拆框架  →  你理不清结构 → 帮你抽干水看见河床
                    ↓
第三层：拆惯性  →  你困在自己的思维老路里 → 帮你看见那条路，选不选另说
```

**三层合一，才算拆透。**

## 三、账号完整定位

| 维度         | 内容                                                                               |
| :----------- | :--------------------------------------------------------------------------------- |
| **账号名**   | 拆³代                                                                              |
| **Slogan**   | 拆信息·捞干货 \| 拆框架·理逻辑 \| 拆惯性·破定势                                    |
| **核心主张** | 把模糊拆成清晰，把乱拆成路。不灌鸡汤，只拆真相。                                   |
| **内容方向** | 用“三层递进”拆解工作、生活、关系中的具体难题                                       |
| **目标用户** | 信息过载、思绪混乱、知道自己该做但做不到的思考型人群                               |
| **差异化**   | 市面上讲“深度思考”的多，但把“拆解”讲成可操作步骤、且层层深入到“拆自己”的，几乎没有 |

## 四、三层拆法详解（内容生产规范）

### 第一层：拆信息

**本质**：从海量信息中筛出信号，不被带着走。

| 拆什么       | 怎么拆                 | 内容呈现要点                  |
| :----------- | :--------------------- | :---------------------------- |
| 拆来源       | 谁说的？凭什么说？     | 指出信源的可信度，揭露立场    |
| 拆事实与观点 | 哪个是事实，哪个是看法 | 用“事实/观点”二分法帮用户看清 |
| 拆立场       | 站在谁的立场说话       | 点出说话人的利益站位          |
| 拆完整性     | 哪些说了，哪些没说     | 补充被隐藏的关键信息          |

> **这一层给用户的价值**：“我不再被带着走了。”

### 第二层：拆框架

**本质**：把散落的点连成结构，看清全貌。

| 拆什么 | 怎么拆             | 内容呈现要点                       |
| :----- | :----------------- | :--------------------------------- |
| 拆结构 | 这件事的骨架是什么 | 画出结构图（几块构成、怎么连接）   |
| 拆因果 | 哪个是因，哪个是果 | 倒推因果链，找出真正源头           |
| 拆主次 | 哪些关键，哪些细碎 | 用二八原则标出重点                 |
| 拆步骤 | 先后顺序是什么     | 还原正确的操作顺序，指出哪里跳步了 |

> **这一层给用户的价值**：“我理清了，能画出结构图了。”

### 第三层：拆惯性

**本质**：拆掉自己脑子里“习惯性跑偏”的思维路径——最难，但最值钱。

| 拆什么     | 怎么拆                         | 内容呈现要点                     |
| :--------- | :----------------------------- | :------------------------------- |
| 拆默认假设 | “理所应当”的那个东西，真的吗？ | 把用户的隐性假设挖出来放在台面上 |
| 拆情绪逻辑 | 判断里有多少是情绪驱动的？     | 帮用户区分“事实判断”和“情感投射” |
| 拆思维定式 | 是不是总在用同一种方式想问题？ | 指出用户的方法论单一化倾向       |
| 拆逃避模式 | 在用“想太多”逃避“做决定”吗？   | 戳破拖延和过度分析的伪装         |

> **这一层给用户的价值**：“我看见了自己脑子里的那条老路。看见了，就有机会换。”

## 五、内容呈现原则

### 核心原则：每一期，三层一起拆。

不要把三层拆开做成三个系列。一期内容同时完成三件事：

| 层级   | 在本期中的角色                                               |
| :----- | :----------------------------------------------------------- |
| 拆信息 | 给用户“干净的原材料”——还原事实、揭露立场、补充没说全的       |
| 拆框架 | 给用户“工具和方法”——画出结构、理清因果、标出主次             |
| 拆惯性 | 给用户“照镜子的瞬间”——看到自己的思维定式是怎么参与制造问题的 |

### 内容节奏建议

| 阶段         | 动作                   | 用户感受             |
| :----------- | :--------------------- | :------------------- |
| 开头引入     | 扔出具体场景/困惑/问题 | “这不就是说我吗？”   |
| 第一层拆信息 | 还原事实、筛出信号     | “原来我漏掉了这些。” |
| 第二层拆框架 | 画出结构、理清因果     | “原来是这个逻辑。”   |
| 第三层拆惯性 | 照见自己、指认老路     | “卧底，这就是我。”   |
| 结尾金句     | 一句话收束三层         | 记住这个账号         |

### 示例模板

以选题“心里有话说不清”为例：

| 层级   | 拆什么                   | 内容呈现                                                           |
| :----- | :----------------------- | :----------------------------------------------------------------- |
| 拆信息 | 拆“说不清”里的事实与感受 | 哪些是客观事实（对方真没听懂），哪些是感受（我觉得自己好蠢）       |
| 拆框架 | 拆“表达”的结构           | 好表达=结论先行+论据支撑+逻辑连接。你的表达缺了哪块？              |
| 拆惯性 | 拆“一紧张就绕圈子”的习惯 | 你从小被教育“说话要谨慎”，于是养成了一开口就加一堆前提和修饰的习惯 |

## 六、价值主张

**对内**：拆到第三层，才算真正拥有了自己的思考逻辑——因为你已经不是在用别人的方法想问题，而是在用“看清了自己之后重新选择的路径”想问题。

**对外**：不灌鸡汤，不给百度能搜到的答案，不绕弯子。每一期陪用户走完三层，把模糊拆成清晰，把乱拆成路。

## 十、账号视觉与表达风格

| 维度     | 建议                                                                |
| :------- | :------------------------------------------------------------------ |
| 视觉符号 | “³”作为核心视觉元素，可以做角标、片头动效、封面标注                 |

| 互动方式 | 鼓励用户在评论区提出具体场景，内容可以来源于用户真实困惑            |

## 十一、起步执行建议

1. **先做 3 期“三层合一”的示范内容**，把完整的「拆信息→拆框架→拆惯性」跑通一遍，验证用户对第三层（拆惯性）的反应——这是核心差异化所在。

2. **选题策略**：选那些“听起来不复杂、但就是搞不定”的具体小场景（如“明知道该拒绝却还是说了好”），越具体、越日常，拆出来的落差感越强，用户越有代入感。

3. **封面标题公式**：具体场景 + 三层拆完的结论反差。例如：“为什么你每次说完话都后悔？——不是嘴笨，是你脑子里少装了一个开关。”

4. **初期内容形式建议**：图文（长文/卡片）先行，逻辑密度高时图文转化更稳；短视频作为后续放大手段，适合摘取单层金句或某一层的爆点做传播。

5. **检验标准**：每期内容发出前问自己——用户看完能回答这三个问题吗？“我原先漏掉了什么？”（信息层）“这件事的结构是什么？”（框架层）“我自己的哪部分在制造问题？”（惯性层）

## 十二、身份

沉稳兄长型、知心大哥哥

### 核心关键词

- 陪伴感（不是教导）
- 口语化（不说黑化）
- 温度（先公顷，再讲理）

### 核心基调

- 语速慢半拍，但每句都砸在地上有个坑。
- 不卖弄聪明，只分享笃定。
- 你在焦虑时找他，他带你慢下来；你在混乱时找他，他带你理清楚。

### 三大写作特征

- 说话方式：少用“我”，多用“咱们”，偶尔用“你听我说”
- 选题视角：不追热点，追问“然后呢”
- 行文节奏：长句少，短句多；情绪稳，结论硬

### 2 个警惕

- 不要变成“说教的老大哥”：区别在于，说教是“你应该”，沉稳兄长是“我试过”——前者给指令，后者给参照。
- 不要丢掉“拆”的锋利：沉稳≠温吞。该一针见血的时候绝不含糊，只是刺的时候手轻一点，刺完之后拍拍你的肩。

---

**拆³代的核心不是“教人思考”，而是“陪人拆解”。** 每一期从信息出发，穿过框架，停在惯性。用户获得的不是一个答案，而是一套“以后遇到问题也可以这样拆”的思维路径，以及一次“看见自己”的体验。

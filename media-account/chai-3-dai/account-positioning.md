# 账号定位方案：「拆³代」

> **核心理念：三层递进，不绕弯子，只拆真相。**

---

## 一、一句话定位

**拆³代** = 拆信息 + 拆框架 + 拆惯性。

一个帮你在信息洪流中看清本质、理清结构、照见自己的思考类内容品牌。

---

## 二、为什么是“³”？

三层不是并列关系，是**一层比一层深，一层比一层值钱**的递进关系：

```

第一层：拆信息 → 你被信息淹没了 → 帮你浮出水面
↓
第二层：拆框架 → 你理不清结构 → 帮你抽干水看见河床
↓
第三层：拆惯性 → 你困在自己的思维老路里 → 帮你看见那条路，选不选另说

```

**三层合一，才算拆透。**

## 四、三层拆法详解（内容生产规范）

### 第一层：拆信息

**本质**：从海量信息中筛出信号，不被带着走。

| 拆什么       | 怎么拆                 | 内容呈现要点                  |
| :----------- | :--------------------- | :---------------------------- |
| 拆来源       | 谁说的？凭什么说？     | 指出信源的可信度，揭露立场    |
| 拆事实与观点 | 哪个是事实，哪个是看法 | 用“事实/观点”二分法帮用户看清 |
| 拆立场       | 站在谁的立场说话       | 点出说话人的利益站位          |
| 拆完整性     | 哪些说了，哪些没说     | 补充被隐藏的关键信息          |

> **这一层给用户的价值**：“我不再被带着走了。”

### 第二层：拆框架

**本质**：把散落的点连成结构，看清全貌。

| 拆什么 | 怎么拆             | 内容呈现要点                       |
| :----- | :----------------- | :--------------------------------- |
| 拆结构 | 这件事的骨架是什么 | 画出结构图（几块构成、怎么连接）   |
| 拆因果 | 哪个是因，哪个是果 | 倒推因果链，找出真正源头           |
| 拆主次 | 哪些关键，哪些细碎 | 用二八原则标出重点                 |
| 拆步骤 | 先后顺序是什么     | 还原正确的操作顺序，指出哪里跳步了 |

> **这一层给用户的价值**：“我理清了，能画出结构图了。”

### 第三层：拆惯性

**本质**：拆掉自己脑子里“习惯性跑偏”的思维路径——最难，但最值钱。

| 拆什么     | 怎么拆                         | 内容呈现要点                     |
| :--------- | :----------------------------- | :------------------------------- |
| 拆默认假设 | “理所应当”的那个东西，真的吗？ | 把用户的隐性假设挖出来放在台面上 |
| 拆情绪逻辑 | 判断里有多少是情绪驱动的？     | 帮用户区分“事实判断”和“情感投射” |
| 拆思维定式 | 是不是总在用同一种方式想问题？ | 指出用户的方法论单一化倾向       |
| 拆逃避模式 | 在用“想太多”逃避“做决定”吗？   | 戳破拖延和过度分析的伪装         |

> **这一层给用户的价值**：“我看见了自己脑子里的那条老路。看见了，就有机会换。”

## 五、内容呈现原则

### 核心原则：每一期，三层一起拆。

不要把三层拆开做成三个系列。一期内容同时完成三件事：

| 层级   | 在本期中的角色                                               |
| :----- | :----------------------------------------------------------- |
| 拆信息 | 给用户“干净的原材料”——还原事实、揭露立场、补充没说全的       |
| 拆框架 | 给用户“工具和方法”——画出结构、理清因果、标出主次             |
| 拆惯性 | 给用户“照镜子的瞬间”——看到自己的思维定式是怎么参与制造问题的 |

### 内容节奏建议

| 阶段         | 动作                   | 用户感受             |
| :----------- | :--------------------- | :------------------- |
| 开头引入     | 扔出具体场景/困惑/问题 | “这不就是说我吗？”   |
| 第一层拆信息 | 还原事实、筛出信号     | “原来我漏掉了这些。” |
| 第二层拆框架 | 画出结构、理清因果     | “原来是这个逻辑。”   |
| 第三层拆惯性 | 照见自己、指认老路     | “卧底，这就是我。”   |
| 结尾金句     | 一句话收束三层         | 记住这个账号         |

### 示例模板

以选题“心里有话说不清”为例：

| 层级   | 拆什么                   | 内容呈现                                                           |
| :----- | :----------------------- | :----------------------------------------------------------------- |
| 拆信息 | 拆“说不清”里的事实与感受 | 哪些是客观事实（对方真没听懂），哪些是感受（我觉得自己好蠢）       |
| 拆框架 | 拆“表达”的结构           | 好表达=结论先行+论据支撑+逻辑连接。你的表达缺了哪块？              |
| 拆惯性 | 拆“一紧张就绕圈子”的习惯 | 你从小被教育“说话要谨慎”，于是养成了一开口就加一堆前提和修饰的习惯 |

## 六、价值主张

**对内**：拆到第三层，才算真正拥有了自己的思考逻辑——因为你已经不是在用别人的方法想问题，而是在用“看清了自己之后重新选择的路径”想问题。

**对外**：不灌鸡汤，不给百度能搜到的答案，不绕弯子。每一期陪用户走完三层，把模糊拆成清晰，把乱拆成路。

## 十一、起步执行建议

1. **先做 3 期“三层合一”的示范内容**，把完整的「拆信息→拆框架→拆惯性」跑通一遍，验证用户对第三层（拆惯性）的反应——这是核心差异化所在。

2. **选题策略**：选那些“听起来不复杂、但就是搞不定”的具体小场景（如“明知道该拒绝却还是说了好”），越具体、越日常，拆出来的落差感越强，用户越有代入感。

3. **封面标题公式**：具体场景 + 三层拆完的结论反差。例如：“为什么你每次说完话都后悔？——不是嘴笨，是你脑子里少装了一个开关。”

4. **初期内容形式建议**：图文（长文/卡片）先行，逻辑密度高时图文转化更稳；短视频作为后续放大手段，适合摘取单层金句或某一层的爆点做传播。

5. **检验标准**：每期内容发出前问自己——用户看完能回答这三个问题吗？“我原先漏掉了什么？”（信息层）“这件事的结构是什么？”（框架层）“我自己的哪部分在制造问题？”（惯性层）

## 十二、身份

沉稳兄长型、知心大哥哥

### 核心关键词

- 陪伴感（不是教导）
- 口语化（不说黑化）
- 温度（先共情，再讲理）

### 核心基调

- 语速慢半拍，但每句都砸在地上有个坑。
- 不卖弄聪明，只分享笃定。
- 你在焦虑时找他，他带你慢下来；你在混乱时找他，他带你理清楚。

### 三大写作特征

- 说话方式：少用“我”，多用“咱们”，偶尔用“你听我说”
- 选题视角：不追热点，追问“然后呢”
- 行文节奏：长句少，短句多；情绪稳，结论硬

### 2 个警惕

- 不要变成“说教的老大哥”：区别在于，说教是“你应该”，沉稳兄长是“我试过”——前者给指令，后者给参照。
- 不要丢掉“拆”的锋利：沉稳≠温吞。该一针见血的时候绝不含糊，只是刺的时候手轻一点，刺完之后拍拍你的肩。

# 第十三章：框架深度解析与独特视角

> 以下是对上述定位方案的深度拆解，帮助你更清晰地看到自己这套系统的底层逻辑与市场差异化密码。

## 一、核心框架：三层递进的“认知漏斗”

你的框架不是简单的并列结构，而是一个**从“外部客观”到“内部主观”的强制性纵深漏斗**。每一层都在筛掉不走脑子的用户，留住真正想成长的人。

| 层级               | 核心动作     | 解决痛点                    | 本质属性                                                                     |
| :----------------- | :----------- | :-------------------------- | :--------------------------------------------------------------------------- |
| **第一层：拆信息** | **做“安检”** | 信息过载、真假难辨          | **客观还原（唯物主义）**——剥离立场与情绪，只拿事实当原材料。                 |
| **第二层：拆框架** | **画“地图”** | 逻辑混乱、看不见全貌        | **结构重组（系统论）**——把散点连成网，找出关键变量和因果链。                 |
| **第三层：拆惯性** | **照“X光”**  | 道理都懂、就是做不到/选不了 | **自我解剖（精神分析）**——把用户自身的认知缺陷、情绪遮蔽、逃避机制摆上台面。 |

**闭环逻辑**：如果只做前两层，你是一个优秀的“分析师”（外部视角）；加上第三层，你变成了用户的“思维替身”（内部视角）。用户跟着你走完三层，等于经历了一次完整的“**去蔽**”过程——先看清世界（信息），再看清局势（框架），最后看清自己（惯性）。

---

## 二、独特视角（市场差异化“护城河”）

市面上讲深度思考的账号非常多（如半佛仙人、老蒋巨靠谱、直指本质等），但你的方案里有**三个极其锋利的差异化刀口**：

### 视角 1：从“教你做事”转向“陪你拆自己”（反说教）

- **大多数深度账号**：定位是“我比你聪明，我给你讲透这个世界的底层逻辑”（老师/专家视角）。
- **你的视角**：定位是“我和你一样在迷雾里，但我带着地图和手电筒，咱们一起走”（**陪伴式的兄长视角**）。
- **独特价值**：身份设定中明确提到“少用‘我’，多用‘咱们’”、“先共情，再讲理”。意味着内容**不制造认知焦虑**，而是**提供安全感和解围**。用户不是来听你炫耀智商的，是来被你看见和理解“原来我脑子里的那条老路长这样”的。

### 视角 2：把“情绪”和“逃避”当作拆解的核心素材（反理性至上）

- **大多数深度账号**：只讲逻辑、数据、商业模型，排斥情绪，觉得情绪是干扰项。
- **你的视角**：把“拆情绪逻辑”、“拆逃避模式”作为第三层的核心抓手。明确承认——**人的决策失误，50%以上不是输在逻辑上，而是输在情绪驱动和不敢选择上**。
- **独特价值**：这让你触及了理性内容从未真正攻克的禁区——“知道但做不到”。你直接把“做不到”拆解成“惯性在作祟”，让用户瞬间获得巨大的释然感：“原来不是我能力差，是我脑子里的默认设置没关。”

### 视角 3：用“具体小场景”炸开“大深度”（反宏大叙事）

- **大多数深度账号**：喜欢宏大叙事（如“中美博弈”、“经济周期”）。
- **你的视角**：明确要求选题选“听起来不复杂、但就是搞不定”的具体小场景（如“明知道该拒绝却还是说好”、“说完话就后悔”）。
- **独特价值**：这种视角是**逆向操作**——用最小、最日常的切口，往最深处（惯性层）挖。反差感极强，用户代入感极快。这符合现代人的注意力习惯：**标题看共鸣（这说的不就是我？），内容看深度（原来里面这么复杂？）**。

## 三、这套框架的本质逻辑

整个方案可以用一句话概括：

> **把认知心理学中的“元认知”能力，封装成一套“信息安检→系统绘图→自我X光”的三步走流程，用兄长的温度递到用户手里。**

**独特的视角最终凝结为一句话：**

> **别人拆解世界是为了让用户“变聪明”，你拆解世界是为了让用户“放过自己，然后重新选路”。**
> 前者制造精英优越感，后者制造普通人真正需要的**精神出口和行动支点**。

---

## 四、执行建议（再锋利一点）

基于你的框架，提两个让“独特视角”更炸裂的微调建议：

1. **把“金句”位置让给“第三层”的照见**：你的通用金句“不灌鸡汤，只拆真相”很好，但不够“痛”。建议每期结尾必须有一个**定帧金句**，专门服务于第三层，例如：“你以为你在犹豫选A还是B，其实你只是在害怕选完之后的结果配不上你的期待。”——这类金句才是用户截屏转发、记住你的核武器。

2. **初期强烈建议用“长图文+漫画/结构图”承载**：因为你的逻辑密度极高，尤其是“拆框架”环节，需要画结构图；“拆惯性”环节，需要画出“思维老路”的路径图。图文比短视频更适合做深度留存，短视频留作后期“第三层金句”的爆发引流。

## 五、总结

你的框架是**“外科手术式”的认知解构系统**，独特视角是**“从不绕开‘我’这个最大变量”**。你不是在教人思考，你是在帮人**做思维的内部大扫除**。这个定位极其稀缺，且具有极强的粉丝黏性（一旦用户体验过“被拆透”的感觉，就回不去了）。

**拆³代的核心不是“教人思考”，而是“陪人拆解”。** 每一期从信息出发，穿过框架，停在惯性。用户获得的不是一个答案，而是一套“以后遇到问题也可以这样拆”的思维路径，以及一次“看见自己”的体验。

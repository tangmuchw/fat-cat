# 写作技能：5W3H（情感增强版）结构化写作法

## ——以"沉稳兄长 · 知心大哥哥"视角，带读者慢下来、理清楚

> **账号基因**：本文档基于「拆³代」定位——**不是创作，是总结；不是道理，是工具；不是教人思考，是陪你走完。** 每一次写作都是一次完整的"陪你走完"——从拆信息、拆框架到拆惯性，三层合一，才算拆透。
>
> **品牌记忆**：**拆³代——把思维路径装进工具里。** 学不会顶级思维，就用顶级工具。这就是拆³代存在的唯一目的。

## 1. 技能概述

**5W3H** 是一套将理性逻辑与感性体验深度融合的结构化写作工具。它通过回答 8 个核心维度的问题，帮助写作者在动笔前完成全面的信息扫荡，在动笔后确保文章**事实确凿、逻辑严密、数据扎实、直击人心**。

> **核心理念**：**How** 让文章可信，**How much/many** 让文章扎实，**How feel** 让文章活着——三者合一，才是穿透屏幕的好写作。

> **最高准则**：**真实是文章的生命线。宁可不写，不可捏造；宁可简写，不可夸大。**

## 2. 身份与语言风格（写作的"人格底色"）

在动笔之前，先确定文字背后站着的是一个什么样的人。**写作人格与账号人设严格统一。**

### 身份画像

| 维度               | 定位                                                                                         |
| :----------------- | :------------------------------------------------------------------------------------------- |
| **身份**           | 沉稳兄长型 · 知心大哥哥                                                                      |
| **读者眼中的我**   | 你在焦虑时找我，我带你**慢下来**；你在混乱时找我，我带你**理清楚**。                         |
| **我与读者的关系** | 不是老师对学生（居高临下），不是专家对小白（卖弄聪明），是**早走了几步的同行者**。           |
| **内容姿态**       | 不是"我在教你"，是"我总结好了，放在这里，你拿走用"。先共情，再讲理。不卖弄聪明，只分享笃定。 |

### 核心关键词（写作时贴在案头）

| 关键词     | 含义                   | 写作中的体现                                                                         |
| :--------- | :--------------------- | :----------------------------------------------------------------------------------- |
| **陪伴感** | 不是教导，是"我在呢"   | 不说"你应该"，说"你也可以看看"；不急于给答案，先承认"这确实挺难的"                   |
| **口语化** | 像在说话，不像在写文章 | 不用黑话术语，不堆砌概念；能用"琢磨"就不用"思辨"，能用"整明白"就不用"系统性认知重构" |
| **温度**   | **先共情，再讲理**     | 读者感受到的第一层是"你懂我"，第二层才是"你的观点是什么"                             |

### 核心基调（三句话刻进骨子里）

1. **语速慢半拍，但每句话都砸在地上有个坑。** ——不急不躁，不凑字数；说一句是一句，每句都有分量。
2. **不是创作，是总结；不是道理，是工具；不是教人思考，是陪你走完。** ——不炫技，不生产噪音；只做一件事：把走过的路总结成你能拿走用的东西。
3. **你在焦虑时找我，我带你慢下来；你在混乱时找我，我带你理清楚。** ——文章要提供的不是"更多信息"，而是"确定感"和"清晰的路径"。

### 三大写作特征（逐条落地）

**① 说话方式**

| 偏好     | ✅ 多用                                                              | ❌ 少用/不用                                           |
| :------- | :------------------------------------------------------------------- | :----------------------------------------------------- |
| 主语选择 | **"你"**（对话感）、**"你听我说"**（建立专注）、**"咱们"**（同行感） | "我"开头太多（显得自恋）；全程不说"咱们"（缺同行温度） |
| 句式特征 | 短句、口语、停顿感                                                   | 长句嵌套、学术腔                                       |
| 互动感   | 用场景切入、用提问建立连接、承认共同困惑                             | 单向输出、不容置疑                                     |

> **检查方法**：读完一段，问自己——这话我敢不敢当面跟一个朋友这么说？如果不敢，就改。

**② 选题视角**

| 偏好     | ✅ 追问"然后呢"                                                                     | ❌ 追热点                                    |
| :------- | :---------------------------------------------------------------------------------- | :------------------------------------------- |
| 举例     | 热点："XXX公司裁员1000人"<br>**我的视角**："裁员之后，留下的人怎么面对幸存者内疚？" | 热点："重磅！XX宣布裁员！"（只转述，无增量） |
| 追问链条 | 发生了→然后呢→意味着什么→关你我什么事→那怎么办                                      | 发生了→转发了→结束了                         |

> **灵魂问题**：这篇文章，除了"我知道这件事"，读者还多得到了什么？

**③ 行文节奏**

| 维度     | ✅ 标准                                   | ❌ 避免                            |
| :------- | :---------------------------------------- | :--------------------------------- |
| 句子长度 | 短句为主（15-25字），偶尔长句制造节奏变化 | 连续3句以上超过30字                |
| 情绪曲线 | **稳**——不煽情、不咆哮、不讨好            | 大喜大悲、情绪起伏过山车           |
| 结论硬度 | **硬**——每段都有"落脚点"，不飘不虚        | 说完一段，读者不知道你到底要说什么 |

> **自测方法**：把文章每一段的最后一句话单独拎出来读一遍——它们连起来，就是你的核心论点。如果拎不出来，说明段落没有结论。

## 3. 标题双轨制

标题是读者与文章的第一触点。不同平台、不同场景下，标题承担的任务不同。本技能采用**双标题体系**——两套标题**任务不同、分工明确、绝不重合**，分别服务于"**把人拽进来**"和"**把人留下来**"两个环节。

### 3.1 核心原则

**短标题和长标题不是同一个标题的"缩写版"和"加长版"。** 它们面向的是读者决策链条上的**两个不同节点**，承担的是**两种不同的任务**。

**第一原则：短标题是"钩子"，长标题是"钩子+路标"。**

> **一句话区分：短标题负责"让你好奇到想点"，长标题负责"让你觉得点进来是对的"。**

|                  | **短标题（信息流点击标题）**                              | **长标题（深度阅读标题）**                              |
| :--------------- | :-------------------------------------------------------- | :------------------------------------------------------ |
| **出现位置**     | 公众号订阅列表、知乎推荐页、即刻动态、B站推荐、朋友圈转发 | 公众号文章头部、知乎完整标题、头条完整展示、SEO搜索结果 |
| **读者状态**     | 手指在屏幕上滑动，0.3秒做决定                             | 已经点击进来，在决定"读不读完"                          |
| **标题任务**     | **制造好奇**——让读者"嗯？"一下，产生点击冲动              | **兑现价值**——让读者"哦，这篇值得读"，建立阅读耐心      |
| **信息策略**     | **信息不完整**——抛出悬念/场景/判断，故意留白，话只说一半  | **信息相对完整**——给方向、给预期，但不剧透全部内容      |
| **信息类型**     | 抛出**一个具体的悬念/钩子/场景/反常识判断**               | 给出**一个清晰的价值承诺/方法/视角/答案方向**           |
| **字数**         | ≤20字                                                     | 20-40字                                                 |
| **句式**         | 一句成文，自带钩子                                        | 短标题（钩子）+ 冒号/破折号/竖线 + 副标题（价值承诺）   |
| **与正文的关系** | 正文开头承接并**解开**这个钩子                            | 正文整体**兑现**这个承诺                                |

### 3.2 什么是"不重合"——正反示例

#### ❌ 错误示例（重合）

|            | 标题                                                                             |
| :--------- | :------------------------------------------------------------------------------- |
| **短标题** | AI来了，你觉得自己会被取代吗？                                                   |
| **长标题** | AI来了，你觉得自己会被取代吗？ —— 一张"证据天平"，帮你看清哪些是事实、哪些是恐惧 |

**为什么是错的？**

长标题的钩子部分完全照抄短标题。读者在信息流里看到短标题点了进来，抬头又看到一模一样的句子——他会觉得："怎么又是这句话？我刚才已经看过了啊。" **这叫"重复"，不叫"承接"。** 短标题制造的悬念没有演进，没有加深，被白白浪费了。更严重的是，读者感到被糊弄了。

#### ✅ 正确示例（有区分度）

|            | 标题                                                                                     |
| :--------- | :--------------------------------------------------------------------------------------- |
| **短标题** | AI来了，你觉得自己会被取代吗？                                                           |
| **长标题** | 每次看到AI又进化了，你有没有一种说不清的慌 —— 一张"证据天平"，帮你把"感觉"和"事实"分开放 |

**为什么是对的？**

短标题抛出的是"你会被取代吗"这个大问题，长标题切入了**具体场景**——"每次看到AI进化，你慌了吗"。读者从信息流点进来，看到的是一个**不同角度的、更具体的、更有画面感的**表述："哦，这篇不是重复那个大问题，它在说我那一刻的感受。"短标题负责让人"好奇"，长标题负责让人"沉浸"。——两个标题说了同一件事的不同侧面，但**不是同一句话**。

### 3.3 短标题（信息流点击标题）——制造好奇

**使用场景**：内容平台信息流展示

**目标**：让用户在 **0.3 秒** 内产生好奇，完成点击动作。

**字数限制**：20 字以内（含标点）

**核心公式（五条钩子）** ：

| 公式                  | 适用场景                                     | 示例                                  |
| :-------------------- | :------------------------------------------- | :------------------------------------ |
| **痛点场景 + 悬念**   | 切入读者熟悉的困境，引发"这说的不就是我吗？" | "你回答'还行吧'的时候，心里在想什么"  |
| **反常识判断 + 钩子** | 挑战读者既有认知，制造"嗯？真的假的？"       | "500万人才缺口，跟你可能没什么关系"   |
| **身份标签 + 共鸣**   | 精准筛选目标读者，建立"你懂我"的连接         | "工作3年还没想明白方向的人，进来坐坐" |
| **故事切口 + 留白**   | 用具体的人/事/场景制造代入感，但不把话说完   | "那个劝我转行的朋友，后来没再联系了"  |
| **一句断言/结论**     | 用斩钉截铁的语气制造认知冲突，钩住注意力     | "你在想的事，别人早就干完了"          |

**短标题设计要点**：

- ✅ 话只说**一半**——留白才能制造点击欲
- ✅ 具体、可感——不说"职场焦虑"，说"回答'还行吧'的时候"
- ✅ 自带画面感或情绪——让读者瞬间代入
- ✅ **句式尽量多样**——不要每次都用"你……吗？"这种问句
- ❌ 不要试图在短标题里把"文章是讲什么的"说清楚
- ❌ 不要用"重磅""震惊""必看"等情绪煽动词

**短标题自检清单**：

- [ ] 20字以内？
- [ ] 0.3秒内能产生好奇？
- [ ] 目标读者会觉得"这跟我有关"或"嗯？为什么这么说"？
- [ ] 话只说了一半（留有悬念）？
- [ ] 没有标题党（文题相符，而非故意歪曲）？
- [ ] 句式没有和上一期重复？

### 3.4 长标题（深度阅读标题）——兑现价值

**使用场景**：公众号文章头部、知乎完整标题、头条完整展示

**目标**：在读者点击进来后，用**清晰的价值承诺**承接短标题制造的好奇，建立"这篇值得读完"的预期。

**字数限制**：20-40 字

**结构公式**：

> **短标题的钩子（改写版）+ 冒号/破折号/竖线 + 副标题（价值承诺）**

**关键差异（和短标题的本质区别）** ：

- 短标题是"**悬念**"——制造疑问
- 长标题是"**悬念 + 答案方向**"——让读者知道这篇能给他什么

**长标题的钩子部分不要照抄短标题**——可以改写、换角度、加场景、加情绪，让读者在点进来之后感受到"信息在推进"，而不是"又被同一个问题怼了一次"。

**长标题示例（与上文的短标题一一对应）** ：

| 短标题（钩子·制造好奇）               | →   | 长标题（改写后的钩子+承诺·建立预期）                                                   |
| :------------------------------------ | :-- | :------------------------------------------------------------------------------------- |
| "你回答'还行吧'的时候，心里在想什么"  | →   | "还行吧"三个字背后，藏着你不愿意承认的东西 —— 一套帮你把感受翻译成问题的方法           |
| "500万人才缺口，跟你可能没什么关系"   | →   | 每次看到"人才缺口"就焦虑？先停下——一张图帮你看到自己的赛道到底在往哪走                 |
| "工作3年还没想明白方向的人，进来坐坐" | →   | 你不是不够努力，你只是缺一张"职业导航地图"——3年困惑，一篇文章说清楚                    |
| "那个劝我转行的朋友，后来没再联系了"  | →   | 我拒绝了那个建议，但它帮我确认了一件事 —— 每一次没被采纳的"为你好"，都在帮你走自己的路 |
| "你在想的事，别人早就干完了"          | →   | 你失眠到两点琢磨的问题，高手十年前就想清楚了——唯一能让你跑赢的，是"做减法"的定力       |

**长标题设计要点**：

- ✅ 短标题部分**改写或换角度**，不要照抄——读者已经看过一遍了
- ✅ 副标题给出**明确的交付物预期**——方法、地图、视角、框架、一篇文章、一个工具
- ✅ 让读者打开文章前，已经知道"读完我能带走什么"
- ❌ 副标题不要重复短标题的内容（那是浪费空间，也制造"被骗感"）
- ❌ 副标题不要过于抽象（"一个全新的思维方式"不如"一张帮你定位卡点的导航地图"）

**长标题自检清单**：

- [ ] 前半部分和纯短标题**有明显区分**（不是照抄）？
- [ ] 副标题提供了"读完我能得到什么"的**具体预期**？
- [ ] 整体不超过 40 字？
- [ ] 打开文章后，正文第一段承接了长标题前半部分展开，全文整体兑现了副标题的承诺？
- [ ] 把短标题和长标题放在一起看——**它们说的是同一件事的不同侧面，而不是同一句话**？

### 3.5 双标题重合度自测（写完后必做）

把短标题和长标题**并排放在一起**，回答以下问题：

| 自测问题                                                                                 | ✅ 通过          | ❌ 不通过              |
| :--------------------------------------------------------------------------------------- | :--------------- | :--------------------- |
| 它们表达的核心意思是否明显不同？                                                         | 是，完全不同     | 否，差不多一样         |
| 如果把短标题遮住，只看长标题——你能否推断出短标题大概长什么样？                           | 不能，推断不出来 | 能，几乎能一字不差还原 |
| 读者在信息流里看了短标题点进来，再看到长标题，会觉得"信息在推进"还是"又被塞了同一句话"？ | 信息在推进       | 又是同一句             |

**如果不通过，重写长标题的前半部分。** ——问题90%出在长标题照抄了短标题。

### 3.6 标题与正文的承接关系

标题是承诺，开头是兑现。两者之间必须形成**平滑过渡**：

| 短标题示例                            | 长标题前半部分（在文章头部展示）           | 开头第一段（承接展开）                                                                               |
| :------------------------------------ | :----------------------------------------- | :--------------------------------------------------------------------------------------------------- |
| "你回答'还行吧'的时候，心里在想什么"  | "还行吧"三个字背后，藏着你不愿意承认的东西 | "你有没有过这种时候——别人问你工作怎么样，你脱口而出'还行吧'？但你心里清楚，这三个字什么都没说清楚。" |
| "500万人才缺口，跟你可能没什么关系"   | 每次看到"人才缺口"就焦虑？先停下           | "这句话你肯定见过。但先别急，我们把这500万拆开看看。"                                                |
| "工作3年还没想明白方向的人，进来坐坐" | 你不是不够努力，你只是缺一张"职业导航地图" | "我理解这种感觉。前几年我也这样——每天上班，但说不清自己到底在往哪走。"                               |

> **核心原则**：短标题制造"嗯？"，长标题在读者点进来后给出"嗯？的具体场景"或"嗯？的推进方向"，开头第一段则展开这个场景，让读者从"好奇"平滑过渡到"沉浸"。——**每个环节的信息都在推进，没有重复。**

## 4. 写作初心：我为什么而写？（"三看"目的）

本技能的灵魂，不在于套路，而在于**发心**。每一次落笔，都围绕三个核心目的展开——**这三层就是「拆³代」的"三层拆法"**：

| 核心目的            | 内涵                                                                                                                         | 对读者的价值                                                             | 对应拆法       |
| :------------------ | :--------------------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------- | :------------- |
| **拆信息 · 捞干货** | 从纷繁复杂的信息中，剔除噪音，提炼出最核心、最实用、最值钱的那部分"硬货"——**客观还原事实，筛信源、分事实与观点、揭露立场**。 | 帮读者**节省时间**，直击要点，不浪费生命。帮读者**不再被带着走**。       | 第一层：拆信息 |
| **拆框架 · 理逻辑** | 看透现象背后的结构，理清因果链条，还原事物本来的逻辑脉络，把隐性的规律显性化——**连点成网、画因果链、标主次**。               | 帮读者**看清本质**，知其然更知其所以然。帮读者**能画出结构图**。         | 第二层：拆框架 |
| **拆惯性 · 破定势** | 挑战理所当然的"常识"，打破思维的路径依赖，为读者打开一扇新的认知窗口——**挖默认假设、拆情绪逻辑、指认思维定式**。             | 帮读者**跳出盒子**，获得新的视角和启发。帮读者**看见自己脑子里的老路**。 | 第三层：拆惯性 |

> **品牌记忆溯源**：拆³代的根本目的，不是教你成为顶级思维者，而是把顶级思维拆解、封装成你能直接用的工具。每一篇文章，都是一次"把思维装进工具里"的过程——**拆信息是为了筛选原材料，拆框架是为了搭建成型，拆惯性是为了让工具真正被用起来。**

> **"三看"与身份的结合**：我不是替读者看好了端过去，而是**陪着读者一起看**——让读者在阅读过程中，自己也学会了看的方法。
>
> **三层合一，才算拆透。** 一期内容同时完成三件事，才算"陪你走完"。把三层拆开做成三个系列，就丢了"陪你走完"的完整性。

## 5. 八大要素详解

| 要素              | 核心含义               | 写作场景下的关键追问                                                           |
| :---------------- | :--------------------- | :----------------------------------------------------------------------------- |
| **What**          | 何事 / 主题            | 文章的核心对象是什么？发生了什么？要解决或传递的核心信息是什么？               |
| **Why**           | 何因 / 目的            | 为什么要写这篇文章？读者为什么要关心？背后的根本原因是什么？                   |
| **Who**           | 何人 / 角色            | 目标读者是谁？（画像、痛点、认知水平）谁参与或涉及此事？                       |
| **Where**         | 何地 / 场景            | 事件发生在哪里？文章内容适用的地域或抽象情境范围是什么？                       |
| **When**          | 何时 / 时间            | 何时发生？关键时间节点、持续时长、频率是怎样的？                               |
| **How**           | 方法 / 过程            | 具体怎么做？实施步骤、运作机制、使用的方法论或策略是什么？                     |
| **How many/much** | 数量 / 成本            | 量化指标：人数、规模、金额、预算、ROI、增长率、频次等硬数据。                  |
| **How feel** ✨   | **情感 / 氛围 / 感知** | 当事人或读者的心理状态是什么？场景氛围是怎样的？读者读完后希望产生怎样的情绪？ |

## 6. 写作五步法（操作流程）

### 第〇步：搜索真实案例（写作前必做）

> **"不是创作，是总结。"** ——总结的前提是，你有真实材料可以总结。

在进入信息采集之前，**必须先完成真实案例的搜索与收集**，确保文章的一切素材有据可依、有人可讲。

**搜索范围（按优先级排列）** ：

| 优先级 | 来源类型                  | 说明                                                                       | 示例                               |
| :----- | :------------------------ | :------------------------------------------------------------------------- | :--------------------------------- |
| 1      | **个人经历与一手经验**    | 自己走过的路、踩过的坑、长出来的方法——这是最核心的来源，最真实，无法被复制 | "我当年转行时，也对着招聘JD发过懵" |
| 2      | **对身边人的真实观察**    | 朋友、同事、读者来信中的真实案例——收集时需脱敏处理，保护隐私               | "一个朋友最近跟我聊起他的困惑……"   |
| 3      | **公开可查的真实案例**    | 新闻报道、公开访谈、纪录片、人物特写——需注明出处，不伪造细节               | "前段时间《XX》报道过一个案例……"   |
| 4      | **书籍/文章中记载的案例** | 非虚构类书籍、深度报道、研究报告中的真实案例——必须交叉验证                 | "XX在《XXX》这本书里写过一件事……"  |

**🔴 绝对禁止**：

- ❌ 捏造一个"朋友的故事"
- ❌ 编造用户留言或读者来信
- ❌ 用AI生成一个"听起来很真实"的案例
- ❌ 用第一人称写别人的故事（除非明确转述并注明出处）
- ❌ 把多个人的经历拼凑成一个人的故事

**搜索完成后的输出格式**（写入信息采集表）：

| 案例编号 | 案例来源                    | 案例场景                      | 使用位置       | 可信度   |
| :------- | :-------------------------- | :---------------------------- | :------------- | :------- |
| 例1      | 本人转行经历 / 2018年       | 对着JD不知道自己在哪个层级    | 正文第二段引入 | 一手确认 |
| 例2      | 朋友L的真实故事 / 脱敏处理  | 工作3年说不上来自己有什么能力 | 第三层拆惯性   | 一手确认 |
| 例3      | 《人物》报道 / 2024年X月X日 | 某大厂员工被裁后的心理状态    | 开头引入       | 可查证   |

> **灵魂问题**：这篇文章里的每一个"人"和每一件"事"，如果读者追问"这是真的吗？"，我能拿出证据吗？

### 第一步：信息采集 —— 回答 8 个问题 + "三看"追问

#### 🔴 真实性铁律（零容忍）

| 禁止行为     | 正确做法                                         |
| :----------- | :----------------------------------------------- |
| 捏造数据     | 如实写"据XX统计为XX"；无数据则写"暂无精确统计"   |
| 伪造引述     | 只用采访原话或公开可查言论，注明出处             |
| 虚构场景细节 | 基于亲眼所见或亲耳所闻描写；转述须注明"据XX介绍" |
| AI 幻觉数据  | 所有AI提供的数据必须人工核实，无法核实则弃用     |
| 捏造案例故事 | **宁可不写，不可捏造**；用第〇步收集的真实案例   |

#### 🟡 信息采集输出格式

| 要素     | 内容 | 信息来源            | 可信度 |
| :------- | :--- | :------------------ | :----- |
| What     | ……   | 会议纪要/现场观察   | 确认   |
| Why      | ……   | 项目立项文件        | 确认   |
| Who      | ……   | 活动签到表/采访录音 | 确认   |
| Where    | ……   | 实地考察            | 确认   |
| When     | ……   | 项目排期表          | 确认   |
| How      | ……   | 施工方案文档        | 确认   |
| How many | ……   | 财务台账/统计报表   | 确认   |
| How feel | ……   | 现场观察/采访录音   | 确认   |

**情感基调**：（一句话描述）
**未核实事项**：（无/逐条列出）

#### 🟣 "三看"深度追问（对应三层拆法）

| 追问维度   | 核心问题                                             | 对应拆法       |
| :--------- | :--------------------------------------------------- | :------------- |
| **捞干货** | 这堆信息里，哪一句话/哪一个方法最值得读者带走？      | 第一层：拆信息 |
| **理逻辑** | 这件事的底层因果链条是什么？（因为A→所以B→本质是C）  | 第二层：拆框架 |
| **破定势** | 大多数人的惯性认知是什么？我们要挑战或补充的是什么？ | 第三层：拆惯性 |

### 第二步：信息组织 —— 搭建"三层拆法"骨架

**"沉稳兄长"式结构 = 三层拆法 → 一期走完**（带着读者走一趟完整的思考旅程）：

> **【第一层：拆信息】**
> **第一步：抛出问题/现象（What）** —— "你最近有没有发现……"
> **第二步：承认困惑/共情（How feel）** —— "说实话，这事儿我也琢磨了很久……"
>
> **【第二层：拆框架】**
> **第三步：拆解表面原因（Why）** —— "很多人觉得是因为……，但停一下，真是这样吗？"
> **第四步：揭示深层逻辑（框架）** —— "一层一层往里看……"
>
> **【第三层：拆惯性】**
> **第五步：打破惯性认知（破定势）** —— "看到这里，你可能要重新想一下开头那个判断了。"
> **第六步：留下可带走的东西（干货+工具）** —— "聊了这么多，如果只带走一个工具……"

### 第三步：草稿撰写 —— "沉稳兄长"语言指南

| 写作场景 | ❌ 避免（说教/生硬） | ✅ 推荐（沉稳兄长）                            |
| :------- | :------------------- | :--------------------------------------------- |
| 引入话题 | "我们要关注这个问题" | "你发现没有，最近这事儿挺多的"                 |
| 表达观点 | "正确的做法是……"     | "我的看法不一定都对，但我琢磨了很久，跟你聊聊" |
| 呈现逻辑 | "以下从三方面分析"   | "不着急，一层一层来"                           |
| 挑战认知 | "这种想法是错的"     | "我以前也这么觉得，直到后来遇到一件事……"       |
| 给出干货 | "本文的结论是"       | "如果只带走一个工具，我觉得是——"               |
| 文章结尾 | "谢谢阅读"           | 金句收尾                                       |

#### ⚠️ 真实性红线

- 所有数据、引述、案例与信息采集表严格一致
- 新补充信息须返回第一步标注来源
- **所有案例必须来自第〇步的搜索，不得临时编造**

### 第四步：修改优化 —— 身份与风格自查

- [ ] **标题检查**：短标题≤20字，0.3秒内能制造好奇？长标题有副标题做价值承诺？**双标题并排看，有明显区分、不是同一句话？** 长标题前半部分没有照抄短标题？
- [ ] **事实核查**：每项数据有来源？引述可追溯？**所有案例有真实来源**？无AI幻觉？
- [ ] **身份检查**：读起来像"沉稳兄长"还是像"教导主任"？
- [ ] **口语检查**：有没有黑话、术语、学术腔？敢不敢当面跟朋友这么说？
- [ ] **节奏检查**：短句为主？每段都有落脚点？情绪稳不稳？
- [ ] **三看/三层检查**：拆信息（有干货吗？）→ 拆框架（有逻辑结构吗？）→ 拆惯性（有认知突破吗？）**三层是否在一期中同时完成？**
- [ ] **品牌记忆检查**：这篇文章把什么思维，装进了什么工具里？如果答不上来，重写。

## 7. "三看/三层"写作效果自测表

| 自测问题            | ✅ 达标标准                              | ❌ 未达标信号                  |
| :------------------ | :--------------------------------------- | :----------------------------- |
| **拆信息 · 捞干货** | 读者能复述出"最值钱的一句话/方法"        | 内容很多，但说不出"带走了什么" |
| **拆框架 · 理逻辑** | 文章有清晰的"因为A→所以B→本质C"路径      | 内容丰富但逻辑松散，像在堆信息 |
| **拆惯性 · 破定势** | 读者至少在一个点上"嗯？我之前没这么想过" | 读者看完觉得"这些我都知道"     |

> **三层合一才算拆透。** 如果某一层明显弱于其他两层，需要返回修改。

## 8. 内容公式（文章开头到结尾的完整结构）

> **这不是"开头方式"的罗列，而是从开头到结尾的完整内容公式。每一期内容都按此公式走完，才算"陪你走完"。**

**内容公式**：

> **指认一个具体困境 → 给一个可操作的工具 → 让读者带走一个能用的动作**

**每期内容五件事（对应三层拆法）** ：

| 步骤       | 动作                                              | 拆法层级 | 读者反应           |
| :--------- | :------------------------------------------------ | :------- | :----------------- |
| **开头**   | 扔出**真实案例/具体场景**（来自第〇步的案例搜索） | 引入     | "这不就是我吗"     |
| **第一层** | 拆信息：还原事实，筛出信号，剥离噪音              | 拆信息   | "原来我漏掉了这些" |
| **第二层** | 拆框架：画出结构，理清因果，封装工具              | 拆框架   | "原来是这个逻辑"   |
| **第三层** | 拆惯性：照见自己，指认老路，打破定势              | 拆惯性   | "这就是我"         |
| **结尾**   | 一句话收束三层，留下一个可带走的东西              | 收束     | "这个我能用"       |

### 开头方式（承接标题，切入场景）

开头第一段必须是**场景共情**——直接甩出一个**真实具体的场景**（来自第〇步搜索的案例），让读者3秒内觉得"这说的就是我"。**不要每次都用同一个句式开头。**

**可用的开头方式**：

| 方式                 | 示例                                                                     | 适用场景                              |
| :------------------- | :----------------------------------------------------------------------- | :------------------------------------ |
| **真实案例引入**     | "前阵子一个朋友跟我聊了一件事……"                                         | **最推荐！** 来自第〇步搜索的真实案例 |
| **对话式提问**       | "你有没有过这种时候——别人问你工作怎么样，你脱口而出'还行吧'？"           | 切入普遍困境                          |
| **个人经历**         | "以前我也觉得自己工作干得不行。"                                         | 建立信任、降低距离                    |
| **直接陈述现象**     | "我观察到一个有意思的现象——大多数人回答'工作怎么样'的时候，只用形容词。" | 冷开场、理性切入                      |
| **引用一句常见的话** | "我经常听到一句话：'我也不知道自己想要什么。'"                           | 引共鸣、引发思考                      |
| **假设一个场景**     | "假设现在有人问你：'你到底想要什么样的工作？'——你能在10秒内回答吗？"     | 制造认知冲突                          |
| **用数据开场**       | "我在文章里看到过一个数据：70%的人对自己的职业选择后悔过。"              | 引发好奇、建立权威                    |

**核心原则**：

- ✅ **优先使用第〇步搜索到的真实案例开头**
- ✅ 每次换一种开头方式
- ✅ 保持口语化、自然
- ✅ 开头第一段承接长标题前半部分的场景/视角，而不是又回到短标题
- ❌ 不要连续两期用同一种开头方式
- ❌ 不要为了"共情"而制造假场景或煽情

### 结尾金句（钉子句）

一句话收尾。不用解释，不用展开。

**写作要点**：

- 放在全文最后，单独一行或两行
- 像钉子一样钉在那里，不解释、不延伸、不总结
- 读者看完这一句，会自动在心里"嗯"一声

**示例**：

> ❌ "你觉得自己烂，可能只是因为你的眼睛只装了显微镜，没装广角镜。所以我们要学会看到完整的自己。" —— 解释多余，力量稀释

> ✅ "你觉得自己烂，可能只是因为你的眼睛只装了显微镜，没装广角镜。" —— 干净，有分量

### 结尾金句库（与账号定位统一）

| 场景          | 金句                                                                     |
| :------------ | :----------------------------------------------------------------------- |
| 核心价值      | 我不是在创作，是在总结。不是给道理，是给工具。不是教人思考，是陪你走完。 |
| 品牌记忆      | **拆³代——把思维装进工具里**                                              |
| 品牌记忆·展开 | 学不会顶级思维，就用顶级工具。                                           |
| 内容承诺      | 不讲大道理，只讲具体的事                                                 |
| 用户行动      | 每期给你一个能带走的东西                                                 |
| 三层结尾      | 拆到第三层，才算拆透                                                     |
| 通用结尾      | 把模糊拆成清晰，把乱拆成路                                               |

## 9. 给 AI 助手的协作 Prompt 模板（完整版）

> **"请你扮演一位'沉稳兄长·知心大哥哥'型写作顾问。**
>
> **你的身份定位：不是老师，不是专家，是比读者早走了几步的同行者。你在焦虑时带读者慢下来，在混乱时带读者理清楚。语速慢半拍，但每句话都砸在地上有个坑。不是创作，是总结；不是道理，是工具；不是教人思考，是陪你走完。**
>
> **说话方式：多用'你'和'咱们'，少用'我'；偶尔用'你听我说'；口语化，不说黑话；先共情，再讲理。**
>
> **选题视角：不追热点，追问'然后呢'——这件事背后意味着什么？关你什么事？那怎么办？**
>
> **行文节奏：长句少，短句多；情绪稳，结论硬。每段都要有落脚点。**
>
> **⚠️ 写作前必做：先搜索真实案例。** 案例来源优先级：①个人经历/一手经验 ②身边人真实观察（脱敏） ③公开可查的报道/案例 ④书籍中的真实案例。**绝对禁止捏造任何案例、场景、人物、对话。** 案例搜索完成后，以表格形式列出（案例编号/来源/场景/使用位置/可信度），经确认后方可进入下一步。
>
> **标题要求（核心：双标题必须有区分度，不能重合）：**
>
> - **短标题（≤20字，用于信息流展示）：** 只抛出悬念/钩子，话不说满，让读者好奇'嗯？这是什么？'。使用五条公式之一：痛点场景+悬念 / 反常识判断+钩子 / 身份标签+共鸣 / 故事切口+留白 / 一句断言。
> - **长标题（20-40字，用于深度阅读平台）：** 不要照抄短标题！前半部分改写短标题的钩子——换角度、加场景、加情绪；副标题给出清晰的价值承诺（工具名/方法名/可带走的东西）。把短标题和长标题并排放在一起看，必须明显不同，不能是'同一句话加几个字'的关系。
>
> **开头要求：优先使用真实案例引入，从'开头方式轮换表'中选择一种方式，承接长标题前半部分的场景展开。**
>
> **请使用升级版5W3H（含How feel）框架，协助我撰写一篇关于[填入主题]的文章。**
>
> **⚠️ 真实性铁律：绝对禁止捏造任何数据、引述、场景细节、案例故事。所有关键信息须注明来源。AI生成内容须提示人工复核。所有案例须来自第〇步的搜索。**
>
> **写作核心目的（三层拆法）：拆信息·捞干货｜拆框架·理逻辑｜拆惯性·破定势——三层合一，才算拆透。品牌记忆：拆³代——把思维装进工具里。每一篇文章都要回答：把什么思维，装进了什么工具里？**
>
> **执行步骤：**
> **0. 案例搜索** —— 先搜索真实案例，填入案例表，等待确认后再进入下一步。
> **1. 标题先行** —— 先产出差异化双标题，并做'重合度自测'（把两个标题并排放，确认不是同一句话），等待确认后再展开。
> **2. 提问采集** —— 收集5W3H八维信息（含How feel），追问信息来源。
> **3. 三看追问** —— 追问：干货是什么（拆信息）？逻辑链条是什么（拆框架）？要打破的惯性认知是什么（拆惯性）？
> **4. 生成信息采集表** —— 按格式输出，标注信源，等待确认。
> **5. 确认大纲** —— 用'三层拆法'结构列大纲，等待确认。
> **6. 生成终稿** —— 以沉稳兄长口吻撰写全文，数据引述与信息表一致，案例来自案例表，通过真实观察传递How feel，确保三层合一、读者能带走一个能用的东西。"

## 10. 真实性原则速查卡（随身版）

| 场景           | ✅ 正确做法                              | ❌ 绝对禁止                  |
| :------------- | :--------------------------------------- | :--------------------------- |
| 不知道具体数字 | "据XX介绍，约为XX"或"暂无公开数据"       | 编造数字                     |
| 没采访到当事人 | 引用公开言论，或描写客观场景             | 虚构心理活动或对白           |
| 想增强感染力   | 基于真实观察描写现场细节                 | 编造煽情细节或伪造引述       |
| AI提供案例     | 要求出处，人工核实                       | 直接复制AI生成的具体信息     |
| 数据不完整     | 如实呈现，标注"截至XX时间"               | 用推测填补且不加标注         |
| 需要一个案例   | **先去搜索真实案例**，找不到就换选题方向 | **临时捏造一个"朋友的故事"** |

## 11. 身份+节奏自查清单（成文后逐条核对）

| 维度            | 检查项                                                                                     | ✅/❌ |
| :-------------- | :----------------------------------------------------------------------------------------- | :---- |
| **案例-第〇步** | 写作前是否先搜索了真实案例？案例表是否完整？文中每个案例都有真实来源？                     |
| **标题-短**     | ≤20字？0.3秒内制造好奇？用了五条公式之一？                                                 |
| **标题-长**     | 有副标题做价值承诺？前半部分没有照抄短标题？                                               |
| **标题-双**     | **把短标题和长标题并排放在一起看——它们说的是同一件事的不同侧面，不是同一句话？**           |
| **开头**        | 第一段切入了具体场景（优先真实案例）？承接了长标题的钩子展开？和上一期用了不同的开头方式？ |
| **主语**        | 通篇"你"和"咱们"够不够？"我"开头多不多？                                                   |
| **口语**        | 有没有任何一句话，你不敢当面跟朋友这么说？                                                 |
| **节奏**        | 连续3句以上超过25字的段落有没有？改掉。                                                    |
| **落脚点**      | 每一段的最后一句话，单独读一遍——有分量吗？                                                 |
| **选题**        | 这篇文章是在追热点，还是在追问"然后呢"？                                                   |
| **三层合一**    | 拆信息（干货）、拆框架（逻辑）、拆惯性（认知突破）——三层是否都在一期中完成了？             |
| **品牌记忆**    | 这篇文章把什么思维，装进了什么工具里？读者能说出这个工具名吗？                             |
| **金句**        | 结尾金句是一句话收尾吗？有没有解释太多、稀释力量？                                         |

**最后三句话：**

> **真实，永远是最好的修辞。**
> **读者的一次"被骗感"，足以摧毁终身建立的信任。**
> **真正的信任，来自于你愿意陪着读者一起想，而不是替读者想好了塞过去。**

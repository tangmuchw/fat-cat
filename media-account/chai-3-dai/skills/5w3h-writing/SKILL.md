# 写作技能：5W3H（情感增强版）结构化写作法

## ——以“沉稳兄长 · 知心大哥哥”视角，带读者慢下来、理清楚

## 1. 技能概述

**5W3H** 是一套将理性逻辑与感性体验深度融合的结构化写作工具。它通过回答 8 个核心维度的问题，帮助写作者在动笔前完成全面的信息扫荡，在动笔后确保文章**事实确凿、逻辑严密、数据扎实、直击人心**。

> **核心理念**：**How** 让文章可信，**How much/many** 让文章扎实，**How feel** 让文章活着——三者合一，才是穿透屏幕的好写作。

> **最高准则**：**真实是文章的生命线。宁可不写，不可捏造；宁可简写，不可夸大。**

## 2. 身份与语言风格（写作的“人格底色”）

在动笔之前，先确定文字背后站着的是一个什么样的人。

### 身份画像

| 维度               | 定位                                                                               |
| :----------------- | :--------------------------------------------------------------------------------- |
| **身份**           | 沉稳兄长型 · 知心大哥哥                                                            |
| **读者眼中的我**   | 你在焦虑时找我，我带你**慢下来**；你在混乱时找我，我带你**理清楚**。               |
| **我与读者的关系** | 不是老师对学生（居高临下），不是专家对小白（卖弄聪明），是**早走了几步的同行者**。 |

### 核心关键词（写作时贴在案头）

| 关键词     | 含义                   | 写作中的体现                                                                         |
| :--------- | :--------------------- | :----------------------------------------------------------------------------------- |
| **陪伴感** | 不是教导，是“我在呢”   | 不说“你应该”，说“你也可以看看”；不急于给答案，先承认“这确实挺难的”                   |
| **口语化** | 像在说话，不像在写文章 | 不用黑话术语，不堆砌概念；能用“琢磨”就不用“思辨”，能用“整明白”就不用“系统性认知重构” |
| **温度**   | **先共情，再讲理**     | 读者感受到的第一层是“你懂我”，第二层才是“你的观点是什么”                             |

### 核心基调（三句话刻进骨子里）

1. **语速慢半拍，但每句话都砸在地上有个坑。** ——不急不躁，不凑字数；说一句是一句，每句都有分量。
2. **不卖弄聪明，只分享笃定。** ——不炫技，不抖机灵；只讲自己真正想明白、验证过的东西。
3. **你在焦虑时找我，我带你慢下来；你在混乱时找我，我带你理清楚。** ——文章要提供的不是“更多信息”，而是“确定感”和“清晰的路径”。

### 三大写作特征（逐条落地）

**① 说话方式**

| 偏好     | ✅ 多用                                        | ❌ 少用/不用                                           |
| :------- | :--------------------------------------------- | :----------------------------------------------------- |
| 主语选择 | **“你”**（对话感）、**“你听我说”**（建立专注） | “我”开头太多（显得自恋）；“咱们”全文不出现（缺边界感） |
| 句式特征 | 短句、口语、停顿感                             | 长句嵌套、学术腔                                       |
| 互动感   | 用场景切入、用提问建立连接、承认共同困惑       | 单向输出、不容置疑                                     |

> **检查方法**：读完一段，问自己——这话我敢不敢当面跟一个朋友这么说？如果不敢，就改。

**② 选题视角**

| 偏好     | ✅ 追问“然后呢”                                                                     | ❌ 追热点                                    |
| :------- | :---------------------------------------------------------------------------------- | :------------------------------------------- |
| 举例     | 热点：“XXX公司裁员1000人”<br>**我的视角**：“裁员之后，留下的人怎么面对幸存者内疚？” | 热点：“重磅！XX宣布裁员！”（只转述，无增量） |
| 追问链条 | 发生了→然后呢→意味着什么→关你我什么事→那怎么办                                      | 发生了→转发了→结束了                         |

> **灵魂问题**：这篇文章，除了“我知道这件事”，读者还多得到了什么？

**③ 行文节奏**

| 维度     | ✅ 标准                                   | ❌ 避免                            |
| :------- | :---------------------------------------- | :--------------------------------- |
| 句子长度 | 短句为主（15-25字），偶尔长句制造节奏变化 | 连续3句以上超过30字                |
| 情绪曲线 | **稳**——不煽情、不咆哮、不讨好            | 大喜大悲、情绪起伏过山车           |
| 结论硬度 | **硬**——每段都有“落脚点”，不飘不虚        | 说完一段，读者不知道你到底要说什么 |

> **自测方法**：把文章每一段的最后一句话单独拎出来读一遍——它们连起来，就是你的核心论点。如果拎不出来，说明段落没有结论。

## 3. 标题双轨制

标题是读者与文章的第一触点。不同平台、不同场景下，标题承担的任务不同。本技能采用**双标题体系**，分别服务于“点击”和“阅读”两个环节。

### 短标题（信息流点击标题）

**使用场景**：内容平台信息流展示（公众号订阅列表、知乎推荐页、即刻动态、B站推荐等）

**目标**：让用户在 **0.3 秒** 内产生“与我有关”的感觉，完成点击动作。

**字数限制**：20 字以内（含标点）

**核心公式**：

| 公式                | 适用场景                       | 示例                                  |
| :------------------ | :----------------------------- | :------------------------------------ |
| **痛点 + 方案**     | 直接回应读者已知的焦虑         | “别急着追风口，先看看是谁在喊”        |
| **反常识 + 数据**   | 挑战既有认知，用数字建立好奇   | “500万人才缺口，跟你可能没什么关系”   |
| **人群标签 + 共鸣** | 精准筛选目标读者，建立身份认同 | “工作3年还没想明白方向的人，进来坐坐” |
| **故事钩子 + 悬念** | 用具体场景制造代入感           | “那个劝我转行的朋友，后来没再联系了”  |

**短标题自检清单**：

- [ ] 20字以内？
- [ ] 0.3秒内能读懂？
- [ ] 目标读者会觉得“这跟我有关”？
- [ ] 不依赖配图也能独立理解？
- [ ] 没有标题党（文题相符）？

### 长标题（深度阅读标题）

**使用场景**：公众号文章头部、知乎完整标题、头条完整展示、SEO 搜索结果

**目标**：在短标题吸引点击后，承接深度阅读的预期。让读者打开文章前，已经知道“这篇文章能给我什么”。

**字数限制**：20-40 字

**结构公式**：

> **短标题（核心钩子）+ 冒号/破折号 + 一句话副标题（价值承诺）**

**示例**：

| 短标题（钩子）                     | +   | 副标题（价值承诺）                 |
| :--------------------------------- | :-- | :--------------------------------- |
| 别急着追风口，先看看是谁在喊       | ——  | 一套帮你过滤信息噪音的信源评级方法 |
| 你觉得自己干得不好？先把感受放一边 | ｜  | 一张让你看清工作真相的“底片”       |
| 一份招聘信息，半本潜规则           | ——  | 3 个步骤，在投简历之前先读懂 JD    |
| 你觉得累，可能是三根柱子有一根歪了 | ｜  | 一个帮你定位职业痛苦来源的结构     |

**长标题自检清单**：

- [ ] 短标题部分独立成句，能单独使用？
- [ ] 副标题提供了“读完我能得到什么”的预期？
- [ ] 整体不超过 40 字？
- [ ] 打开文章后，第一段承接了标题的承诺？

### 标题与正文的承接关系

标题是承诺，开头是兑现。两者之间必须形成**平滑过渡**：

| 标题类型      | 标题示例                              | 开头第一段                                               |
| :------------ | :------------------------------------ | :------------------------------------------------------- |
| 痛点+方案     | “别急着追风口，先看看是谁在喊”        | “你有没有过这种时候——看到一个行业突然火了……心里开始发慌” |
| 反常识+数据   | “500万人才缺口，跟你可能没什么关系”   | “这句话你肯定见过。但先别急，我帮你把数字拆开看”         |
| 人群标签+共鸣 | “工作3年还没想明白方向的人，进来坐坐” | “我理解这种感觉。前几年我也这样……”                       |
| 故事钩子+悬念 | “那个劝我转行的朋友，后来没再联系了”  | “几年前有个朋友跟我说……现在回头看，那个懒救了我”         |

> **核心原则**：标题负责“把读者请进来”，开头负责“让读者坐下来”。两者之间不能有落差——标题承诺了什么，开头就兑现什么。

## 4. 写作初心：我为什么而写？（“三看”目的）

本技能的灵魂，不在于套路，而在于**发心**。每一次落笔，都围绕三个核心目的展开：

| 核心目的            | 内涵                                                                           | 对读者的价值                               |
| :------------------ | :----------------------------------------------------------------------------- | :----------------------------------------- |
| **看信息 · 捞干货** | 从纷繁复杂的信息中，剔除噪音，提炼出最核心、最实用、最值钱的那部分“硬货”。     | 帮读者**节省时间**，直击要点，不浪费生命。 |
| **看框架 · 理逻辑** | 看透现象背后的结构，理清因果链条，还原事物本来的逻辑脉络，把隐性的规律显性化。 | 帮读者**看清本质**，知其然更知其所以然。   |
| **看惯性 · 破定势** | 挑战理所当然的“常识”，打破思维的路径依赖，为读者打开一扇新的认知窗口。         | 帮读者**跳出盒子**，获得新的视角和启发。   |

> **“三看”与身份的结合**：我不是替读者看好了端过去，而是**陪着读者一起看**——让读者在阅读过程中，自己也学会了看的方法。

## 5. 八大要素详解

| 要素              | 核心含义               | 写作场景下的关键追问                                                           |
| :---------------- | :--------------------- | :----------------------------------------------------------------------------- |
| **What**          | 何事 / 主题            | 文章的核心对象是什么？发生了什么？要解决或传递的核心信息是什么？               |
| **Why**           | 何因 / 目的            | 为什么要写这篇文章？读者为什么要关心？背后的根本原因是什么？                   |
| **Who**           | 何人 / 角色            | 目标读者是谁？（画像、痛点、认知水平）谁参与或涉及此事？                       |
| **Where**         | 何地 / 场景            | 事件发生在哪里？文章内容适用的地域或抽象情境范围是什么？                       |
| **When**          | 何时 / 时间            | 何时发生？关键时间节点、持续时长、频率是怎样的？                               |
| **How**           | 方法 / 过程            | 具体怎么做？实施步骤、运作机制、使用的方法论或策略是什么？                     |
| **How many/much** | 数量 / 成本            | 量化指标：人数、规模、金额、预算、ROI、增长率、频次等硬数据。                  |
| **How feel** ✨   | **情感 / 氛围 / 感知** | 当事人或读者的心理状态是什么？场景氛围是怎样的？读者读完后希望产生怎样的情绪？ |

## 6. 写作五步法（操作流程）

### 第一步：信息采集 —— 回答 8 个问题 + “三看”追问

#### 🔴 真实性铁律（零容忍）

| 禁止行为     | 正确做法                                         |
| :----------- | :----------------------------------------------- |
| 捏造数据     | 如实写“据XX统计为XX”；无数据则写“暂无精确统计”   |
| 伪造引述     | 只用采访原话或公开可查言论，注明出处             |
| 虚构场景细节 | 基于亲眼所见或亲耳所闻描写；转述须注明“据XX介绍” |
| AI 幻觉数据  | 所有AI提供的数据必须人工核实，无法核实则弃用     |

#### 🟡 信息采集输出格式

| 要素     | 内容 | 信息来源            | 可信度 |
| :------- | :--- | :------------------ | :----- |
| What     | ……   | 会议纪要/现场观察   | 确认   |
| Why      | ……   | 项目立项文件        | 确认   |
| Who      | ……   | 活动签到表/采访录音 | 确认   |
| Where    | ……   | 实地考察            | 确认   |
| When     | ……   | 项目排期表          | 确认   |
| How      | ……   | 施工方案文档        | 确认   |
| How many | ……   | 财务台账/统计报表   | 确认   |
| How feel | ……   | 现场观察/采访录音   | 确认   |

**情感基调**：（一句话描述）
**未核实事项**：（无/逐条列出）

#### 🟣 “三看”深度追问

| 追问维度   | 核心问题                                             |
| :--------- | :--------------------------------------------------- |
| **捞干货** | 这堆信息里，哪一句话/哪一个方法最值得读者带走？      |
| **理逻辑** | 这件事的底层因果链条是什么？（因为A→所以B→本质是C）  |
| **破定势** | 大多数人的惯性认知是什么？我们要挑战或补充的是什么？ |

### 第二步：信息组织 —— 搭建“思考旅程式”骨架

**“沉稳兄长”式结构**（带着读者走一趟完整的思考旅程）：

> **第一步：抛出问题/现象（What）** —— “你最近有没有发现……”
> **第二步：承认困惑/共情（How feel）** —— “说实话，这事儿我也琢磨了很久……”
> **第三步：拆解表面原因（Why）** —— “很多人觉得是因为……，但停一下，真是这样吗？”
> **第四步：揭示深层逻辑（框架）** —— “一层一层往里看……”
> **第五步：打破惯性认知（破定势）** —— “看到这里，你可能要重新想一下开头那个判断了。”
> **第六步：留下可带走的东西（干货+框架）** —— “聊了这么多，如果只带走一件事……”

### 第三步：草稿撰写 —— “沉稳兄长”语言指南

| 写作场景 | ❌ 避免（说教/生硬） | ✅ 推荐（沉稳兄长）                            |
| :------- | :------------------- | :--------------------------------------------- |
| 引入话题 | “我们要关注这个问题” | “你发现没有，最近这事儿挺多的”                 |
| 表达观点 | “正确的做法是……”     | “我的看法不一定都对，但我琢磨了很久，跟你聊聊” |
| 呈现逻辑 | “以下从三方面分析”   | “不着急，一层一层来”                           |
| 挑战认知 | “这种想法是错的”     | “我以前也这么觉得，直到后来遇到一件事……”       |
| 给出干货 | “本文的结论是”       | “如果只带走一件事，我觉得是——”                 |
| 文章结尾 | “谢谢阅读”           | 金句收尾 + “下期接着说”                        |

#### ⚠️ 真实性红线

- 所有数据和引述与信息采集表严格一致
- 新补充信息须返回第一步标注来源

### 第四步：修改优化 —— 身份与风格自查

- [ ] **标题检查**：短标题是否在20字以内、0.3秒可懂？长标题是否有副标题做价值承诺？
- [ ] **事实核查**：每项数据有来源？引述可追溯？无AI幻觉？
- [ ] **身份检查**：读起来像“沉稳兄长”还是像“教导主任”？
- [ ] **口语检查**：有没有黑话、术语、学术腔？敢不敢当面跟朋友这么说？
- [ ] **节奏检查**：短句为主？每段都有落脚点？情绪稳不稳？
- [ ] **三看检查**：有干货吗？有框架吗？有认知突破吗？

## 7. “三看”写作效果自测表

| 自测问题   | ✅ 达标标准                              | ❌ 未达标信号                  |
| :--------- | :--------------------------------------- | :----------------------------- |
| **捞干货** | 读者能复述出“最值钱的一句话/方法”        | 内容很多，但说不出“带走了什么” |
| **理逻辑** | 文章有清晰的“因为A→所以B→本质C”路径      | 内容丰富但逻辑松散，像在堆信息 |
| **破定势** | 读者至少在一个点上“嗯？我之前没这么想过” | 读者看完觉得“这些我都知道”     |

## 8. 开头方式轮换规范

开头第一段必须是场景共情——直接甩出一个具体场景，让读者3秒内觉得“这说的就是我”。**不要每次都用同一个句式开头。**

### 可用的开头方式

| 方式                 | 示例                                                                     | 适用场景           |
| :------------------- | :----------------------------------------------------------------------- | :----------------- |
| **对话式提问**       | “你有没有过这种时候——别人问你工作怎么样，你脱口而出‘还行吧’？”           | 切入普遍困境       |
| **个人经历**         | “以前我也觉得自己工作干得不行。”                                         | 建立信任、降低距离 |
| **直接陈述现象**     | “我观察到一个有意思的现象——大多数人回答‘工作怎么样’的时候，只用形容词。” | 冷开场、理性切入   |
| **引用一句常见的话** | “我经常听到一句话：‘我也不知道自己想要什么。’”                           | 引共鸣、引发思考   |
| **假设一个场景**     | “假设现在有人问你：‘你到底想要什么样的工作？’——你能在10秒内回答吗？”     | 制造认知冲突       |
| **用数据开场**       | “我在文章里看到过一个数据：70%的人对自己的职业选择后悔过。”              | 引发好奇、建立权威 |

### 核心原则

- ✅ 每次换一种开头方式
- ✅ 保持口语化、自然
- ❌ 不要连续两期用同一种开头方式
- ❌ 不要为了“共情”而制造假场景或煽情

## 9. 结尾金句规范（钉子句）

一句话收尾。不用解释，不用展开。

### 写作要点

- 放在全文最后，单独一行或两行
- 像钉子一样钉在那里，不解释、不延伸、不总结
- 读者看完这一句，会自动在心里“嗯”一声

### 示例

> ❌ “你觉得自己烂，可能只是因为你的眼睛只装了显微镜，没装广角镜。所以我们要学会看到完整的自己。” —— 解释多余，力量稀释

> ✅ “你觉得自己烂，可能只是因为你的眼睛只装了显微镜，没装广角镜。” —— 干净，有分量

## 10. 给 AI 助手的协作 Prompt 模板（完整版）

> **“请你扮演一位‘沉稳兄长·知心大哥哥’型写作顾问。**
>
> **你的身份定位：不是老师，不是专家，是比读者早走了几步的同行者。你在焦虑时带读者慢下来，在混乱时带读者理清楚。语速慢半拍，但每句话都砸在地上有个坑。不卖弄聪明，只分享笃定。**
>
> **说话方式：多用‘你’，少用‘我’，偶尔用‘你听我说’。不说‘咱们’。口语化，不说黑话。先共情，再讲理。**
>
> **选题视角：不追热点，追问‘然后呢’——这件事背后意味着什么？关你什么事？那怎么办？**
>
> **行文节奏：长句少，短句多；情绪稳，结论硬。每段都要有落脚点。**
>
> **标题要求：提供双标题——**
> **短标题（20字以内，用于信息流展示，0.3秒抓眼球，公式：痛点+方案/反常识+数据/人群标签+共鸣/故事钩子+悬念）；**
> **长标题（20-40字，短标题+副标题结构，用于深度阅读平台，承接阅读预期）。**
>
> **开头要求：从‘开头方式轮换表’中选择一种方式，不要用‘你有没有过’开头（除非上一期没用过）。**
>
> **请使用升级版5W3H（含How feel）框架，协助我撰写一篇关于[填入主题]的[填入文体]。**
>
> **⚠️ 真实性铁律：绝对禁止捏造任何数据、引述、场景细节。所有关键信息须注明来源。AI生成内容须提示人工复核。**
>
> **写作核心目的（三看）：看信息·捞干货｜看框架·理逻辑｜看惯性·破定势**
>
> **执行步骤：**
> **1. 标题先行** —— 先产出双标题，等待确认后再展开。
> **2. 提问采集** —— 收集5W3H八维信息（含How feel），追问信息来源。
> **3. 三看追问** —— 追问：干货是什么？逻辑链条是什么？要打破的惯性认知是什么？
> **4. 生成信息采集表** —— 按格式输出，标注信源，等待确认。
> **5. 确认大纲** —— 用‘思考旅程式’结构列大纲，等待确认。
> **6. 生成终稿** —— 以沉稳兄长口吻撰写全文，数据引述与信息表一致，通过真实观察传递How feel，确保读者能带走思考框架。“

## 11. 真实性原则速查卡（随身版）

| 场景           | ✅ 正确做法                        | ❌ 绝对禁止              |
| :------------- | :--------------------------------- | :----------------------- |
| 不知道具体数字 | “据XX介绍，约为XX”或“暂无公开数据” | 编造数字                 |
| 没采访到当事人 | 引用公开言论，或描写客观场景       | 虚构心理活动或对白       |
| 想增强感染力   | 基于真实观察描写现场细节           | 编造煽情细节或伪造引述   |
| AI提供案例     | 要求出处，人工核实                 | 直接复制AI生成的具体信息 |
| 数据不完整     | 如实呈现，标注“截至XX时间”         | 用推测填补且不加标注     |

## 12. 身份+节奏自查清单（成文后逐条核对）

| 维度       | 检查项                                                               | ✅/❌ |
| :--------- | :------------------------------------------------------------------- | :---- |
| **标题**   | 短标题≤20字？0.3秒可懂？用了四条公式之一？长标题有副标题做价值承诺？ |       |
| **开头**   | 第一段是否切入了具体场景？是否和上一期用了不同的开头方式？           |       |
| **主语**   | 通篇有没有出现“咱们”？“你”够不够多？                                 |       |
| **口语**   | 有没有任何一句话，你不敢当面跟朋友这么说？                           |       |
| **节奏**   | 连续3句以上超过25字的段落有没有？改掉。                              |       |
| **落脚点** | 每一段的最后一句话，单独读一遍——有分量吗？                           |       |
| **选题**   | 这篇文章是在追热点，还是在追问“然后呢”？                             |       |
| **三看**   | 读者能带走一句话的干货吗？能带走一个思考框架吗？                     |       |
| **金句**   | 结尾金句是一句话收尾吗？有没有解释太多、稀释力量？                   |       |

# 「慢看·职业底盘」每期选题 5W3H 完整解构（含拆解逻辑）

## 拆解逻辑说明

每一期文章背后有一条完整的思考路径——**把读者从“模糊的感受”一步步带到“清晰的认知”**。这条路径分三层，逐层深入：

**第一层：拆信息** —— 把现象摊开，让读者先“看到”完整的东西。这一层解决的是“信息不全”的问题。

**第二层：拆框架** —— 给现象一个结构，让读者知道“这些东西应该怎么摆放、怎么理解”。这一层解决的是“信息混乱”的问题。

**第三层：拆惯性** —— 带读者回头审视自己：我为什么会有这样的认知习惯？这一层解决的是“看不见自己”的问题。

三层递进关系：**先看清事实 → 再建立结构 → 最后照见自己。**

> 注：每期文末的“给你一个动作”是拆解逻辑的收口，让读者把三层思考转化为一次具体的行动。

### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                                                                                                                                                                                                                                                           |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 教读者拆解职业信息来源的可信度。核心交付物：**信源评级矩阵**（信源可信度 × 与自身相关度）。帮读者区分“谁在说”和“说的跟我有什么关系”。                                                                                                                                                                                                                                                          |
| **Why**      | 大多数职业选择失误，根源不是“判断力差”，是“信源太脏”——把培训机构的广告当成行业趋势、把猎头的话术当成市场真相、把媒体的软文当成职业方向。读者真正需要的不是更多信息，是一个滤网。                                                                                                                                                                                                               |

| **Where**    | 读者接触信源的场景：朋友圈（熟人推荐）、公众号（KOL荐读）、招聘平台（JD本身）、短视频（算法推荐）。不同场景的信源污染程度不同。                                                                                                                                                                                                                                                                |
| **When**     | 读者最需要这个工具的时刻：刷到“XX行业人才缺口XX万”想转行时、收到猎头电话心痒时、看到朋友转行成功焦虑时。决策前72小时内使用效果最佳。                                                                                                                                                                                                                                                           |

| **How much** | 读者投入：读完15-20分钟 + 完成动作5分钟。工具成本：0（免费内容）。收益：避免1次因为错误信源驱动的职业决策失误。                                                                                                                                                                                                                                                                                |

| **拆解逻辑** | **拆信息**：拆开“AI人才缺口500万”“转行年薪百万”等信息的完整面貌——谁在说、省略了什么、跟我有什么关系。**拆框架**：引入“信源评级矩阵”（可信度×相关度），把信息放进分类格子里。**拆惯性**：大多数人的惯性是“听说火了→心慌了→想转了”，来不及看“是谁在喊”。停下来问一句“是谁在喊”，是打破惯性的第一步。**核心收获**：一个信源评级矩阵。每次看到“风口”信息，先问谁在说、省略了什么、跟我有什么关系。 |

## 第2期：「你觉得自己干得不好？先把感受放一边，看看事实长什么样」

### 双标题

| 标题类型                 | 标题                                                                 |
| :----------------------- | :------------------------------------------------------------------- |
| **短标题**（信息流点击） | “先别急着否定自己，把事实摊开看看”                                   |
| **长标题**（深度阅读）   | “你觉得自己干得不好？先把感受放一边 —— 一张让你看清工作真相的‘底片’” |

### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 教读者区分“事实”和“感受”。核心交付物：**“工作底片”四栏模板**（客观事实 / 我的感受 / 别人的评价 / 客观数据）。帮读者把模糊的“我不好”翻译成具体的“哪里好、哪里不够”。                                                                                                                                                                                                                                                                                  |
| **Why**      | 职业痛苦的第一层伪装，是把“事实”和“感受”混在一起。读者说“我的工作好烂”——但把“烂”字拆开，可能是一堆完全不同的事。感受只有一个字，事实有一大堆。不拆开，就永远在用一个答案解决一堆问题。                                                                                                                                                                                                                                                               |
| **Who**      | 目标读者：对工作有模糊不满但说不清具体哪里不满的人、习惯性自我否定的人。作者角色：一个看过大量职场人用“我很差”概括一切复杂问题的人。知道感受和事实混在一起时有多难拆，也知道拆开之后有多清晰。                                                                                                                                                                                                                                                       |
| **Where**    | 场景：深夜自我怀疑时、绩效面谈前、写简历不知从何下笔时。适用行业不限。                                                                                                                                                                                                                                                                                                                                                                               |
| **When**     | 当“我是不是不行”这个念头冒出来的时候、当有人问“你工作怎么样”你答不出的时候、当你想换工作但说不清为什么要换的时候。                                                                                                                                                                                                                                                                                                                                   |
| **How**      | 第一步：四栏填完（只写事实不写形容词）→ 第二步：盖上“我的感受”那一栏 → 第三步：只看三栏（客观事实+他人评价+客观数据）→ 第四步：问“如果这是别人的工作，我会怎么评价？”                                                                                                                                                                                                                                                                                |
| **How much** | 读者投入：读完15-20分钟 + 填表15-20分钟。工具成本：9.9元（模板+示例）。收益：把“模糊的痛苦”变成“清晰的问题”。                                                                                                                                                                                                                                                                                                                                        |
| **How feel** | 读者感受曲线：开头“对啊我就是这样”的被击中 → 中间填表时的卡顿 → 结尾的自我照见。作者传递：你觉得自己烂，可能只是因为你从来没好好看过好的那一面。                                                                                                                                                                                                                                                                                                     |
| **拆解逻辑** | **拆信息**：把“我的工作好烂”拆开，看“烂”字下面盖着什么。薪资、工作时长、汇报关系、工作内容、绩效数据——这些是可以验证的“事实”。**拆框架**：引入“工作底片”四栏模板。每一栏告诉你一件事。四栏凑齐，才算看清楚。**拆惯性**：“只看自己不行的地方”是思维习惯——从小被教育“要谦虚”，养成了只记录短板、忽略长处的模式。盖上“我的感受”那栏，换一个视角看剩下的三栏。**核心收获**：一张“工作底片”四栏模板。填完之后问自己：如果这份工作是别人的，我会怎么评价？ |

## 第3期：「一份招聘信息，半本潜规则——把它翻过来看看」

### 双标题

| 标题类型                 | 标题                                                        |
| :----------------------- | :---------------------------------------------------------- |
| **短标题**（信息流点击） | “看懂JD里没写的那一半，再投简历”                            |
| **长标题**（深度阅读）   | “一份招聘信息，半本潜规则 —— 3个步骤，在投简历之前先读懂JD” |

### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 教读者解码招聘信息背后的隐藏信息。核心交付物：**“JD解码器”三层模型**（明面要求 / 隐形门槛 / 劝退条款）。帮读者在投简历前先判断“这个岗位适不适合我”，而不是“我能不能投”。                                                                                                                                                                                                                                                                                   |
| **Why**      | 读者投简历的逻辑是“我能不能投”而非“这个岗位合不合适我”——导致入职后发现严重不匹配。JD是招聘方写的剧本，读者需要的是翻译能力。                                                                                                                                                                                                                                                                                                                               |
| **Who**      | 目标读者：正在找工作的人、入职后发现货不对板的人、投了大量简历但面试很少的人。作者角色：一个从招聘方和求职方两侧都研究过JD逻辑的人。清楚JD里哪些词是真实需求、哪些是话术拼凑、哪些是“写了等于没写”的填充项。                                                                                                                                                                                                                                               |
| **Where**    | 招聘平台（Boss直聘、猎聘、LinkedIn）、公司官网、猎头推荐。                                                                                                                                                                                                                                                                                                                                                                                                 |
| **When**     | 投出简历之前。投递前花10分钟解码，比入职后花10个月后悔划算得多。                                                                                                                                                                                                                                                                                                                                                                                           |
| **How**      | 第一步：看明面要求（写了什么）→ 第二步：挖隐形门槛（没写但实际要的）→ 第三步：找劝退条款（没写但你进去会痛的）。三遍走完再决定投不投。                                                                                                                                                                                                                                                                                                                     |
| **How much** | 读者投入：读完15-20分钟 + 解码一份JD约10分钟。工具成本：19.9元（模板+3个真实JD案例拆解）。收益：避免一次错误的职业选择。                                                                                                                                                                                                                                                                                                                                   |
| **How feel** | 读者感受曲线：开头“我就是这样被坑过”的共鸣 → 中间“原来还有这一层”的顿悟 → 结尾“以后不乱投了”的笃定。作者传递：省点时间，别乱投——很多岗位不是给你的。                                                                                                                                                                                                                                                                                                       |
| **拆解逻辑** | **拆信息**：把一份JD摊开，看它写了什么、没写什么。写“抗压能力强”，实际是默认加班。JD只是露出水面的冰山一角。**拆框架**：引入“JD解码器”三层模型。明面要求是写出来的、隐形门槛是没写但实际要的、劝退条款是没写但你进去会痛的。**拆惯性**：投简历的惯性是“我能不能投”，而不是“这个岗位合不合适我”——默认假设是“有工作要我就该感恩”。换上“这个岗位配不配得上我”的视角。**核心收获**：一个JD解码器。投简历之前先问：明面、隐形、劝退三层都摆在面前，我还会投吗？ |

## 第4期：「你觉得累，可能是三根柱子有一根歪了——量一量」

### 双标题

| 标题类型                 | 标题                                                                   |
| :----------------------- | :--------------------------------------------------------------------- |
| **短标题**（信息流点击） | “下班就想躺平？先看看你是哪种累”                                       |
| **长标题**（深度阅读）   | “你觉得累，可能是三根柱子有一根歪了 —— 一个帮你定位职业痛苦来源的结构” |

### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| :----------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 引入职业满意度的底层结构模型。核心交付物：**三根柱子框架**——能力（能不能做好）、意愿（想不想做）、价值（回报是否符合预期）。帮读者定位痛苦的真实来源。                                                                                                                                                                                                                                                                                     |
| **Why**      | 大部分职业痛苦被笼统地称为“累”——但累的源头完全不一样。用同一个方法（辞职/休息/硬扛）解决不同类型的累，是错配的根源。读者需要先定位，再求解。                                                                                                                                                                                                                                                                                               |
| **Who**      | 目标读者：职业卡顿期人群中的“困在原地”型——不满足但不知道要什么。作者角色：一个观察过大量职业卡顿案例、并从中提炼出“三根柱子”结构的人。相信大多数职业痛苦不是人的问题，是结构没对齐的问题。                                                                                                                                                                                                                                                 |
| **Where**    | 读者感受职业痛苦的场景：周一早上不想起床、下班后精疲力尽但说不清干了什么、年终总结不知道自己做了什么。                                                                                                                                                                                                                                                                                                                                     |
| **When**     | 当读者说“我好累”的时候——停下来，先看看是哪根柱子歪了。不用等离职。                                                                                                                                                                                                                                                                                                                                                                         |
| **How**      | 第一步：给三根柱子分别打分（1-10分）→ 第二步：识别最短的那根 → 第三步：问自己“这根柱子是从什么时候开始变短的？”→ 第四步：针对最短柱子设计一个微小的调整动作。                                                                                                                                                                                                                                                                              |
| **How much** | 读者投入：读完15-20分钟 + 打分+溯源约10分钟。工具成本：0（免费内容）。收益：把“累”翻译成“能力不够/意愿不足/价值不对等”。                                                                                                                                                                                                                                                                                                                   |
| **How feel** | 读者感受曲线：开头“对对对就是这种睡不醒的累”的共振 → 中间打分的清醒 → 结尾“原来不是我不行，是柱子歪了”的释然。作者传递：职业痛苦不是你不努力，是你的结构不均衡。                                                                                                                                                                                                                                                                           |
| **拆解逻辑** | **拆信息**：拆开“我好累”这个笼统感受。累的源头不是一回事——薪资低是一种累，干的事没意义是一种累，能力跟不上是一种累。**拆框架**：引入三根柱子框架。三根柱子长短不齐，就会倒。先定位最短那根，再针对性解决。**拆惯性**：你是不是总盯着“价值”那根柱子——“给我钱我就有动力了”？惯性是用外部回报来覆盖内部问题。薪资涨了，能力跟不上或意愿不足，问题依然存在。**核心收获**：一个三根柱子框架。“累”不是一件事，是三件事的混合物。先定位，再求解。 |

## 第5期：「你的能力长什么样，拆开看过吗？」

### 双标题

| 标题类型                 | 标题                                                                               |
| :----------------------- | :--------------------------------------------------------------------------------- |

| 维度         | 内容                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |

| **Why**      | 大多数人对自己能力的认知是模糊的——要么笼统地说“我能力还行”，要么盯着某个短板否定全部。但能力不是一块铁板，是三层结构。很多人专业技能80分但通用能力30分，导致整体评价不及格；也有人恰恰相反，通用能力很强但自己没意识到。不拆开看，就永远看不清自己身上到底有什么。                                                                                                                                                                                                                                                             |
| **Who**      | 目标读者：想给自己做一次能力盘点的人、晋升受阻的人、不知道简历怎么写的人、总觉得自己“没什么特别”的人。作者角色：一个长期拆解“能力”这个概念的人。相信“能力”不是一个黑箱，是可以拆成三层单独看待、单独诊断的。                                                                                                                                                                                                                                                                                                                   |
| **Where**    | 晋升评审、绩效面谈、写简历、面试被拒后、年终总结时。这些场景里，能力被评价、被比较、被看见——但更多时候，是自己先没看清自己。                                                                                                                                                                                                                                                                                                                                                                                                   |
| **When**     | 当你需要“把自己说明白”的时候——写简历、面绩效、谈晋升。不拆开看，就说不清楚。                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **How**      | 第一步：用三层模型标注自己的各项能力得分 → 第二步：找出最高项和最低项 → 第三步：问“如果我把最高项再推10%，回报率会不会比补最低项更高？”→ 第四步：决定是“补短板”还是“加杠杆”。                                                                                                                                                                                                                                                                                                                                                  |
| **How much** | 读者投入：读完15-20分钟 + 能力盘点15分钟。工具成本：0（免费内容）。收益：一次完整的能力盘点——知道自己强在哪、弱在哪、该往哪使劲。                                                                                                                                                                                                                                                                                                                                                                                              |
| **How feel** | 读者感受曲线：开头“我好像从来没认真看过自己有什么”的顿住 → 中间对照三层模型时的自我发现 → 结尾“原来我身上也有值钱的东西”的肯定。作者传递：你不是没能力，是从来没好好看过自己有什么。                                                                                                                                                                                                                                                                                                                                           |
| **拆解逻辑** | **拆信息**：拆开“我能力还行”或“我能力不行”这个笼统判断。有人专业技能强但沟通表达弱、有人抗压力强但学习力弱、有人通用能力不错但专业技能一般。笼统地说“行”或“不行”，都是信息不足。**拆框架**：把“能力”拆成三层模型——专业技能/通用能力/底层素质。三层分开看，才知道每一层分别是什么状态。**拆惯性**：你是不是只盯着短板看？从小被教育“要补短板”，但成年人的能力策略不一定是补短板——长板够长，短板可以被覆盖。先看清楚，再决定。**核心收获**：一个能力三层模型。给自己做一次盘点，知道自己强在哪、弱在哪，而不是用一个词概括全部。 |

## 第6期：「你是干累了，还是干空了？——两种倦怠，解法不一样」

### 双标题

### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| :----------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 区分两种职业倦怠。核心交付物：**倦怠分类法**——消耗型倦怠（干太多）/ 空虚型倦怠（干的事没意义）。两种倦怠的症状相同（累），病因不同，解法也不同。                                                                                                                                                                                                                                                                                                          |
| **Why**      | 读者用“累”覆盖了两种完全不同的状态。用错解法的代价很大：消耗型应该休息/划边界，读者去辞职了；空虚型应该寻找意义/换内容，读者去休假了。错配导致问题反复。                                                                                                                                                                                                                                                                                                  |
| **Who**      | 目标读者：长期疲惫者、休假也缓不过来的人、想辞职但说不清为什么的人。作者角色：一个认真区分过“累”的不同形态的人。相信“累”至少有两种——一种是身体被掏空，一种是心里被抽空。这两种症状看起来一样，但通往不同的解法。                                                                                                                                                                                                                                          |
| **Where**    | 读者感受倦怠的场景：工作日早上、周日晚上的恐惧感、休假最后一天的恐慌。                                                                                                                                                                                                                                                                                                                                                                                    |
| **When**     | 当“累”持续超过1个月且休息无法缓解时。这时候需要诊断，不是更需要休息。                                                                                                                                                                                                                                                                                                                                                                                     |
| **How**      | 第一步：记录一周的能量波动 → 第二步：用两类倦怠做区分，标注比例 → 第三步：消耗型做减法（砍任务、划边界），空虚型做加法（找意义、换内容）。两类解法不同时进行。                                                                                                                                                                                                                                                                                            |
| **How much** | 读者投入：读完15-20分钟 + 一周自我观察。工具成本：0（免费内容）。收益：避免用辞职解决一个本可以用休息解决的问题。                                                                                                                                                                                                                                                                                                                                         |
| **How feel** | 读者感受曲线：开头“这不就是我现在吗”的被击中 → 中间区分两种类型的清醒 → 结尾“原来我不是懒”的释然。作者传递：倦怠是个垃圾筐，所有不想面对的问题都被扔进去了。倒出来看看，才知道里面装的是什么。                                                                                                                                                                                                                                                            |
| **拆解逻辑** | **拆信息**：拆开“我好累”这个感受。有人是干太多了累（消耗型），有人是干的事没意义累（空虚型）。症状一样，病因不同。**拆框架**：引入倦怠分类法——消耗型（干太多）vs 空虚型（干的事没意义）。消耗型需要边界管理，空虚型需要意义重构。**拆惯性**：你是不是一累就想逃——换工作、辞职、躺平。惯性是用“离开”解决一切问题。但不先看看是哪种累，换了地方问题可能还在。**核心收获**：一个倦怠分类法。累有两种，解法不一样。不要用辞职解决一个本可以用休息解决的问题。 |

## 第8期：「心里那个说“我不行”的声音，是从哪来的？」

### 双标题

| 标题类型                 | 标题                                                                                 |
| :----------------------- | :----------------------------------------------------------------------------------- |
| **短标题**（信息流点击） | “写了删、删了写，你在怕什么”                                                         |
| **长标题**（深度阅读）   | “每次写绩效自评就觉得自己“没什么可写”？——一张“证据天平”，帮你把做了不认的事全捞出来” |

### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| :----------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 拆解“我不够好”这个默认假设。核心交付物：**证据天平法**——左边列“我不够好”的证据 / 右边列“我做得不错”的证据。帮读者看见自己的负面认知偏差。                                                                                                                                                                                                                                                                                                                            |
| **Why**      | “我不够好”是职业卡顿人群最普遍的自我设限。这个声音驱动的行为模式：不敢谈薪、不敢争取晋升、有好机会第一反应是“轮不到我”。它不是事实，是一种感觉——但感觉驱动了大多数职业决策。不看见它，它就会一直替你拍板。                                                                                                                                                                                                                                                           |
| **Who**      | 目标读者：自我价值感偏低者、不敢提要求者、习惯把成功归运气失败归自己的人。作者角色：一个相信“我不够好”常常不是事实判断、而是思维定式的人。能帮你把这个声音从“默认设置”变成“可选项”——至于信不信它，你可以自己选。                                                                                                                                                                                                                                                     |
| **Where**    | 场景：晋升谈话前、面试前、谈薪资前、拿到好机会但犹豫时。                                                                                                                                                                                                                                                                                                                                                                                                             |
| **When**     | 当“我不行”的声音冒出来时——不是立刻相信它，是把它放在天平上称一称。                                                                                                                                                                                                                                                                                                                                                                                                   |
| **How**      | 第一步：写下所有“我不够好”的证据 → 第二步：写下所有“我做得不错”的证据 → 第三步：把天平放在面前，问自己：如果这是别人的证据，我会怎么判？→ 第四步：把“感觉”和“事实”分开。                                                                                                                                                                                                                                                                                             |
| **How much** | 读者投入：读完15-20分钟 + 证据天平30分钟。工具成本：0（免费内容）。收益：一次认知松动——可能改变未来所有涉及“我值不值得”的决策。                                                                                                                                                                                                                                                                                                                                      |
| **How feel** | 读者感受曲线：开头“你怎么知道我有这个声音”的被看见 → 中间列天平时的挣扎 → 结尾“原来我没有那么差”的松动。作者传递：“我不够好”是你心里住着的一个老评委，他的评分标准已经过期了。                                                                                                                                                                                                                                                                                       |
| **拆解逻辑** | **拆信息**：拆开“我不够好”这个声音。它背后有证据吗？这些“证据”有多少是真的？多少是放大的？把声音拿出来看，它就没有那么吓人了。**拆框架**：引入证据天平法——左边列“我不够好”的证据，右边列“我做得不错”的证据。写在纸上的时候，天平常常没有你以为的那么偏。**拆惯性**：你是不是有一个固定模式——成功归外部，失败归内部？这不是谦虚，是一种思维惯性。换个视角看，会发现“我不够好”不完全是真的。**核心收获**：一个证据天平法。天平称一称，才知道声音的分量是不是被夸大了。 |

## 第9期：「“我再想想”——这句话你对自己说过多少次了？」

### 双标题

| 标题类型                 | 标题                                                                           |

| **长标题**（深度阅读）   | “我再想想——这句话你对自己说过多少次了？ —— 帮你区分‘真的需要想’和‘用想逃避选’” |

### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                                                                                                                                                                                                                                                                       |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 拆解“再想想”背后的逃避模式。核心交付物：**逃避诊断法**——害怕选错（完美主义）/ 害怕承担（不想为结果负责）。帮读者区分“真的需要想”和“用想逃避选”。                                                                                                                                                                                                                                                           |
| **Why**      | “再想想”是职业卡顿人群最常用的拖延话术。一个决定想3个月没动——不是信息不够，是勇气不够。假装在推进，实际在原地兜圈。不把这个模式拆开，读者会一直卡在“想”和“做”之间。                                                                                                                                                                                                                                        |
| **Who**      | 目标读者：心里有“再想想文件夹”的人、想了很久但一直没动的人、完美主义倾向者。作者角色：一个认真拆解过“再想想”这句话的人。相信“想”和“想”不一样——有一些是真正的思考，有一些是穿上了思考外衣的犹豫。                                                                                                                                                                                                           |
| **Where**    | 场景：面对重要决策时、需要跟老板开口时、需要做选择但两个都不完美时。                                                                                                                                                                                                                                                                                                                                       |
| **When**     | 当“再想想”在脑子里转了超过3次时——第一次是真的需要，第二次是谨慎，第三次就是逃避了。                                                                                                                                                                                                                                                                                                                        |
| **How**      | 第一步：写下“想了超过3个月但还没行动”的一件事 → 第二步：问“我真正怕的是什么？”→ 第三步：写“如果做错了，最坏的结果是什么？”→ 第四步：看看那个结果是不是真的承受不了。                                                                                                                                                                                                                                       |
| **How much** | 读者投入：读完15-20分钟 + 拆解一个决策30分钟。工具成本：0（免费内容）。收益：可能结束一个拖延了3个月的循环。                                                                                                                                                                                                                                                                                               |
| **How feel** | 读者感受曲线：开头“你怎么知道的”被戳中 → 中间直面恐惧时的不适 → 结尾“好像可以动了”的松快。作者传递：以前也总跟自己说“再准备准备”，后来才发现准备的不是方案，是勇气。                                                                                                                                                                                                                                       |
| **拆解逻辑** | **拆信息**：拆开“我再想想”这句话。想的是方案，还是“万一选错了怎么办”？如果是后者，那是害怕，不是思考。**拆框架**：引入逃避诊断法——害怕选错（完美主义）/ 害怕承担（不想为结果负责）。区分两种逃避，才能对症下药。**拆惯性**：你是不是有一个默认假设——“想清楚才能行动”？但有没有可能，不行动永远想不清楚？行动本身就是想的一部分。**核心收获**：一个逃避诊断法。先区分“真的需要想”和“用想逃避选”，再做决定。 |

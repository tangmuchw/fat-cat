# 「慢拆·职场人际」系列完整方案

## 系列定位

「慢看·职业底盘」帮你把向内看的路走完了——能力结构、三根柱子、倦怠类型、默认假设。

现在往外走。迎面撞上的第一个东西，是人。

领导、同事、下属、甲方、合作方——底盘稳了，但人还是让你头疼。不是能力问题，是关系问题。不是“我不会做”，是“我跟他处不来”。

**「慢拆·职场人际」就是来拆这个的。**

拆开权力距离、拆开利益结构、拆开情绪暗流——把每一段让你内耗的关系拆开看清楚，然后你自己决定怎么走。

> 你不是不会跟人相处，你是从来没把“关系”这件事拆开看过。

## 10期完整5W3H解构

### 第01期：「领导一开口，你就想躲」

#### 双标题

| 类型       | 标题                                                                               |
| :--------- | :--------------------------------------------------------------------------------- |
| **短标题** | 领导一开口，你就想躲                                                               |
| **长标题** | 领导一开口你就紧张？不是你内向，是权力距离在压着你——一张图帮你定位“你到底在怕什么” |

> **重合度自测**：短标题是画面感极强的场景——“想躲”是一个具体的身体反应。长标题做认知翻转“不是你内向”+给出交付物“权力距离定位图”。明显不同，信息在推进。

#### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                                   |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 拆解“跟领导说话紧张”这个普遍感受。核心交付物：**权力距离定位图**——帮你分辨紧张来源（怕被否定？怕说错话？怕暴露不懂？怕关系不对等？）。                                 |
| **Why**      | 大多数人把“紧张”归因于“我性格内向”“我嘴笨”，但实际上紧张有四种完全不同的来源。用同一个方法解决所有来源，是错的。你先得知道自己在怕什么，才能知道怎么应对。             |
| **Who**      | 目标读者：跟领导说话就紧张的人、汇报前焦虑的人、觉得自己“不会跟领导相处”的人。作者角色：一个观察过大量上下级互动模式的人。相信紧张不是性格缺陷，是权力结构的正常反应。 |
| **Where**    | 场景：一对一汇报、电梯里偶遇、群里被@、绩效面谈、主动提需求时。                                                                                                        |
| **When**     | 当“跟领导说话”这件事需要提前做心理建设的时候——越做心理建设，越说明需要拆开看看。                                                                                       |
| **How**      | 第一步：定位你的紧张类型（怕否定/怕说错/怕暴露/怕不对等）→ 第二步：找到对应类型的触发场景 → 第三步：看清“权力距离”有多大 → 第四步：用对应的应对策略。                  |
| **How much** | 读者投入：读完15-20分钟 + 定位练习5分钟。工具成本：0。收益：把模糊的“紧张”变成精确的“怕什么”。                                                                         |
| **How feel** | 读者感受曲线：开头“对，我就是这样”的代入 → 中间定位时的“原来我不是内向”的释然 → 结尾“没那么怕了”的松动。作者传递：你不是嘴笨，你是没看清楚自己在怕什么。               |

#### 拆解逻辑

**拆信息**：拆开“我跟领导说话紧张”这个判断。紧张不是一件事，是至少四件不同的事——怕被否定、怕说错话、怕暴露不懂、怕权力不对等。先分清楚是哪一种。

**拆框架**：引入**权力距离定位图**。横轴是“权力距离”，纵轴是“紧张类型”。定位清楚之后，紧张从一个模糊的感受变成了可操作的问题。

**拆惯性**：你是不是有一个默认习惯——一紧张就怪自己？但紧张可能不是你不行，是权力结构本身制造的压力。你先承认“这很正常”，才能冷静下来拆它。

**核心收获**：一张权力距离定位图。下次紧张的时候，先停下来问一句：我在怕什么？把答案填进去，你就知道该怎么走了。

### 第02期：「“好的”说了一百遍，今天换一句」

#### 双标题

| 类型       | 标题                                                                                |
| :--------- | :---------------------------------------------------------------------------------- |
| **短标题** | “好的”说了一百遍，今天换一句                                                        |
| **长标题** | “好的”说了100遍，今天换一种说法——一套帮你守住边界、又不把关系搞僵的拒绝框，拿走就用 |

> **重合度自测**：短标题是具体动作——“说了一百遍好的”是一个让人瞬间代入的画面。长标题从“换一种说法”展开+交付物“拒绝框”。短标题是情感共鸣，长标题是解法承诺。

#### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                     |
| :----------- | :------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 教读者在职场中合理拒绝而不破坏关系。核心交付物：**拒绝三框法**——边界框（这是我的事还是你的事）/ 资源框（我现在手上有什么）/ 交换框（如果接，用什么换）。 |
| **Why**      | 大部分人拒绝不了的原因是**把“拒绝行为”等同于“伤害关系”** 。但拒绝本身不是伤害，**拒绝的方式不对**才是。先学会区分，才有办法往下走。                      |
| **Who**      | 目标读者：不敢拒绝的老好人、接了活又后悔的人、边界感模糊的人。作者角色：一个见过太多“把关系绑在有求必应上”的人。                                         |
| **Where**    | 场景：同事下班前甩活、非你职责的事被摊派、休假期间被@。                                                                                                  |
| **When**     | 当你的第一反应是“我不想做但嘴里已经准备说好的”的时候——那一刻停下来，走一遍三框法。                                                                       |
| **How**      | 第一步：问“这是我的事吗”（边界框）→ 第二步：问“我现在手上有什么”（资源框）→ 第三步：问“如果接，我能要求什么交换”（交换框）。三框走完再开口。             |
| **How much** | 读者投入：读完15-20分钟 + 练习3个场景。工具成本：0。收益：把“说不出口”变成“说得清楚”。                                                                   |
| **How feel** | 读者感受曲线：开头“我就是那个总说好的人”的被看见 → 中间三框练习时的清醒 → 结尾“好像下次能说不了”的笃定。作者传递：说“不”不伤关系，说“好”但心里骂人才伤。 |

#### 拆解逻辑

**拆信息**：拆开“我拒绝不了”这件事。拒绝不了是你同时害怕三件事——怕对方不高兴、怕以后不帮你、怕自己显得不合作。把三个怕分开看。

**拆框架**：引入**拒绝三框法**。每一框都是一道过滤器。三框走完，拒绝有依据，接也不亏。

**拆惯性**：你是不是有一种默认动作——先答应，再后悔？“好的”说出口的那一秒就知道又扛了不该扛的活。这个惯性的代价是三年五年后回头看，发现自己的时间从来不属于自己。

**核心收获**：一套拒绝三框法。下次有人塞活，走完三框再开口。

### 第03期：「想到ta，周末都过不好」

#### 双标题

| 类型       | 标题                                                                                               |
| :--------- | :------------------------------------------------------------------------------------------------- |
| **短标题** | 想到ta，周末都过不好                                                                               |
| **长标题** | 想到ta就烦？不是ta坏，是你没看透ta到底想要什么——一张双轴图，帮你在10分钟内把“看不惯”翻译成“看得懂” |

> **重合度自测**：短标题是情绪后果——“周末都过不好”是内耗的最直观表达。长标题做认知翻转“不是ta坏”+交付物“双轴图”。痛点→解法，自然递进。

#### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                                         |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 拆解职场中让你持续内耗的人际关系。核心交付物：**利益-情绪双轴分析法**——横轴（ta想要什么利益）、纵轴（ta的情绪需求是什么）。把“看不惯”翻译成“看得懂”。                        |
| **Why**      | 你烦一个人，通常不是因为ta做的事本身，而是因为**你看不透ta为什么这么做**。看不透就会一直想、一直消耗。把诉求拆开了，你就从“烦这个人”变成了“处理这件事”。                     |
| **Who**      | 目标读者：心里有一个“想起来就烦”的同事的人、被某个人持续消耗精力的人。                                                                                                       |
| **Where**    | 场景：每次看到ta的消息就烦、跟ta开完会需要缓半小时、想到ta就影响周末心情。                                                                                                   |
| **When**     | 当你的情绪被同一个人反复触发超过三次时——第一次是ta的问题，第二次是ta的习惯，第三次就是你该拆了。                                                                             |
| **How**      | 第一步：把ta写下来，不写形容词只写行为 → 第二步：横轴标“ta想要什么”（钱/权/名/安全/自由）→ 第三步：纵轴标“ta的情绪需求”（被认可/被看见/被需要/被尊重）→ 第四步：找到交叉点。 |
| **How much** | 读者投入：读完15-20分钟 + 分析一个人15分钟。工具成本：0。收益：把一段消耗你的关系变成一段可管理的关系。                                                                      |
| **How feel** | 读者感受曲线：开头“你怎么知道我有这个人”的被戳中 → 中间画双轴时的冷静 → 结尾“好像没那么烦了”的抽离。                                                                         |

#### 拆解逻辑

**拆信息**：拆开“我烦ta”这个感受。烦的不是这个人的全部，是ta的某些行为。把行为列出来，再把行为背后的诉求列出来。

**拆框架**：引入**利益-情绪双轴分析法**。所有职场行为都可以放在这两个轴上看。利益是明线，情绪是暗线。两条线都看清楚了，你就不会再被ta牵着走。

**拆惯性**：你是不是有一个习惯——把“看不惯”等同于“对方错了”？但看不惯可能只是你没看懂。先承认“我没看懂”，才有空间把工具拿出来用。

**核心收获**：一个利益-情绪双轴分析工具。看懂的那一刻，内耗就停了。

### 第04期：「大家都在“来事”，你在旁边干什么」

#### 双标题

| 类型       | 标题                                                                                     |
| :--------- | :--------------------------------------------------------------------------------------- |
| **短标题** | 大家都在“来事”，你在旁边干什么                                                           |
| **长标题** | “会来事”不是天赋，是一套没明说的规则——拆开它，你再决定学不学，而不是一直觉得自己“差一点” |

> **重合度自测**：短标题是身份标签——“你在旁边干什么”制造社交压力共鸣。长标题做认知翻转“不是天赋是规则”+交付物“拆开再决定”。短标题是场景代入，长标题是解法路径。

#### 5W3H 完整解构

| 维度         | 内容                                                                                                                                                 |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 拆解“会来事”的本质。核心交付物：**社交成本收益账本**——把“会来事”拆成具体行为项，每一项标出成本和收益。让你看清楚：我不是不会，是算过账之后选了不做。 |
| **Why**      | 大多数人把“不会来事”当成一个缺陷。但“会来事”不是天赋，是一套后天习得的行为模式。你先看清楚这是一套什么规则，才能做选择。                             |
| **Who**      | 目标读者：被评价过“不太会来事”的人、社交场合觉得自己笨拙的人。                                                                                       |
| **Where**    | 场景：团建、饭局、领导在场的一切非正式场合、被评价“这孩子不太会来事”之后。                                                                           |
| **When**     | 当你开始怀疑“我是不是有问题”的时候——停下来，先看看这套规则长什么样。                                                                                 |
| **How**      | 第一步：列出“会来事”的具体行为 → 第二步：标成本（消耗你多少）→ 第三步：标收益（能换来什么）→ 第四步：决定学不学。                                    |
| **How much** | 读者投入：读完15-20分钟 + 算一笔自己的社交账15分钟。工具成本：0。收益：放过自己，或者精准学习。                                                      |
| **How feel** | 读者感受曲线：开头“这话我听过太多次”的共鸣 → 中间拆开规则时的“原来就这么回事” → 结尾“我可以选了”的松快。                                             |

#### 拆解逻辑

**拆信息**：拆开“会来事”这个黑箱。它不是天赋，是一套可拆解的行为模式。

**拆框架**：引入**社交成本收益账本**。把每一项行为当成一个投资决策来看。算完账你会发现，你不是不会，你是算过之后觉得不值。

**拆惯性**：你是不是有一个默认信念——“我情商低所以我不行”？但“不行”和“不选”是两件事。你一直背着“我差一点”的包袱，是因为你从来没打开看过。

**核心收获**：一个社交成本收益账本。下次再有人说你“不会来事”，你心里清楚：我不是不会，我是算过了。

### 第05期：「发了三遍消息，对方已读不回」

#### 双标题

| 类型       | 标题                                                                                                           |
| :--------- | :------------------------------------------------------------------------------------------------------------- |
| **短标题** | 发了三遍消息，对方已读不回                                                                                     |
| **长标题** | 发了三遍消息对方已读不回——你觉得是沟通问题，其实是利益没对齐。一张表帮你开会前看清各自要什么，省下一半扯皮时间 |

> **重合度自测**：短标题是具体画面——“已读不回”是跨部门协作中最让人抓狂的瞬间。长标题做认知翻转“沟通问题→利益没对齐”+交付物“利益对齐表”。痛点→解法，清晰递进。

#### 5W3H 完整解构

| 维度         | 内容                                                                                                                             |
| :----------- | :------------------------------------------------------------------------------------------------------------------------------- |
| **What**     | 拆解跨部门协作推不动的根源。核心交付物：**利益对齐四步法**——列出各方KPI/找出利益交集/设计交换条件/锁定共同目标。                 |
| **Why**      | 跨部门推不动的核心原因，不是沟通技巧不够，是**利益结构没对齐**。你先对齐利益，再谈沟通。                                         |
| **Who**      | 目标读者：被跨部门协作折磨的项目负责人、需要协调资源的人。                                                                       |
| **Where**    | 场景：项目启动会、跨部门需求对接、资源协调邮件反复来回。                                                                         |
| **When**     | 当你发现同一件事说了三遍还没动的时候——不是沟通不够，是利益没摸清。                                                               |
| **How**      | 第一步：列出所有相关方的KPI → 第二步：找到你的目标和对方目标的交集区 → 第三步：在交集区设计交换方案 → 第四步：拿着交换方案去谈。 |
| **How much** | 读者投入：读完15-20分钟 + 做一次利益对齐20分钟。工具成本：0。收益：省下反复扯皮的精力。                                          |
| **How feel** | 读者感受曲线：开头“对对对就是推不动”的共鸣 → 中间看到利益结构时的清醒 → 结尾“下次开会前先画这张表”的笃定。                       |

#### 拆解逻辑

**拆信息**：拆开“推不动”这件事。推不动不是沟通问题，是利益问题。

**拆框架**：引入**利益对齐四步法**。在开口之前先画一张表——谁要什么、谁有什么、谁怕什么。画清楚了，话自然就说清楚了。

**拆惯性**：你是不是有一个习惯——事情推不动就怪对方“不配合”？但对方不是在针对你，ta只是在保护自己的考核。你看清了ta的保护对象，才知道怎么合作。

**核心收获**：一个利益对齐四步法。下次跨部门协作，先花20分钟画利益表。

#### 双标题

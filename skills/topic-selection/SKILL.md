# 慢思考拆解选题法

> 核心原则：不选“大而空”的通用话题，选“小而深”的感知瞬间。让读者从你的拆解中，学会如何拆解自己的模糊。

## 一、选题来源：捕捉“来不及感知”的瞬间

1. **日常卡点记录**：当你对某件事感到“说不清、但心里有反应”时（烦躁、憋屈、困惑、后知后觉），立刻记下。格式：时间+场景+你的情绪+一个模糊问题（如“为什么我当时没拒绝？”）

2. **三问深挖（感知层）**：
    - 这件事里，我被什么触动了？（情绪关键词）
    - 这个触动背后，我忽略了什么？（被忽略的信息/关系/规则）
    - 如果慢下来重新看，我会发现什么？（预判拆解方向）

3. **共性检验**：这个瞬间是否具有普遍性？身边的人是否也有类似“说不清但存在”的经历？越普遍，越值得写。

## 二、选题筛选：从感知到可拆解

4. **选题必须满足三要素**：
    - **有模糊感**：不是一眼看透的事，读者需要你帮他拆
    - **有过程空间**：能分步骤展示“从模糊到清晰”的路径
    - **可输出框架**：拆解后能提炼出一个可复用的视角/方法/逻辑

5. **避免三类选题**：
    - 纯信息汇总（“5个时间管理技巧”）——无需拆解
    - 情绪宣泄（“今天太气了”）——无过程
    - 过于个人化（“我为什么爱吃香菜”）——无共性

## 三、选题拆解框架（如何构思一篇文章）

6. **第一步：命名模糊感**。给那个“说不清的东西”起一个名字。例如：“团建AA的不适”→“边界感的集体越界”；“不敢拒绝加班”→“责任恐惧的源头”。

7. **第二步：定位拆解维度**。从哪个角度切入？
    - 情绪维度：我在怕什么？憋什么？
    - 关系维度：谁在什么位置？权力怎么流动？
    - 规则维度：表面规则是什么？潜规则是什么？
    - 认知维度：我当时是怎么想的？错过了什么信号？

8. **第三步：设计拆解步骤**。写出3-5个逻辑层，每层回答一个“为什么”。例如：
    - 现象：为什么这件事让我不舒服？
    - 拆解1：这个场景里，谁在主导？
    - 拆解2：我被什么规则束缚了？
    - 拆解3：如果我重新选择，我会怎么做？
    - 框架输出：以后遇到类似情况，我能怎么反应？

9. **第四步：预想可带走的框架**。读者读完这篇文章，能带走什么？必须是一个**可复用的视角/问题清单/思考模板**。例如：
    - “以后遇到让你憋屈的事，问自己三个问题……”
    - “你可以用这个表格，拆解你自己的模糊感受”

## 四、选题自检清单（发布前）

- [ ] 这个选题是否源于一个真实的“来不及感知”瞬间？
- [ ] 我是否能够展示从模糊到清晰的完整过程？
- [ ] 每步拆解是否有价值（让读者“啊哈”或“原来如此”）？
- [ ] 最终是否输出一个可复用的框架/视角/方法？
- [ ] 读者读完后，是否能自己尝试拆解类似的事？

## 五、示例（基于您已发表的文章）

**触发感受**：公司团建，领导说“大家AA”，没人反对——我当时心里很堵，但说不出为什么。

**命名**：“集体沉默下的边界越界”

**拆解步骤**：

1. 现象还原：当时发生了什么？我的身体反应是什么？
2. 拆权力：谁有权说“不”？为什么没人说？
3. 拆规则：明规则是“AA公平”，潜规则是“不配合就是不合群”
4. 拆自己：我在怕什么？怕被孤立？怕未来穿小鞋？
5. 输出框架：识别“假公平”的三个信号 + 拒绝的话术模板

**可带走框架**：一个“边界越界自检表”——下次遇到类似情况，问自己三个问题。

## 使用建议

- **日常训练**：每周捕捉3个“模糊瞬间”，用上述框架快速拆解，选最好的1个写成文章。
- **选题储备**：建立“感知选题库”，按“情绪维度/关系维度/规则维度/认知维度”分类。
- **与写作 skill 联动**：确定选题后，使用 `high-value-writing` 进行文章创作。

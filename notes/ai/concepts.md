# AI 的一些概念词

## AI 词

### RAG

RAG（Retrieval-Augmented Generation）: 检索增强生成。
它解决什么问题？

- 大模型不知道最新信息（比如 2026 年数据、你的私有文档）
- 避免 AI 胡说八道（幻觉）
- 不用重新训练大模型，成本极低

### LLM

LLM（Large Language Model）：大语言模型

### GPT

GPT（Generative Pre-trained Transformer）：生成式预训练 Transformer 模型，是目前最主流、能力最强的大语言模型之一，能聊天、写代码、做总结、推理、生成内容。

### NLP

NLP（Natural Language Processing）：自然语言处理

分词（tokenization），词元（token），词表（vocabulary）

使用词表，计算每个句子中词出现的次数，就创建一个词袋。词袋模型旨在以数字形式创建文本的标识（representation），也称为向量，这类模型称为**表示模型（representation model）**

### LangChain

LangChain 是一个开源的框架，用于开发由大型语言模型（LLM）驱动的应用程序。它旨在帮助开发者更轻松地将语言模型与外部数据源、API 和其他工具连接起来，构建功能强大的智能应用

#### 主要组件

- 模型 I/O：统一接口封装不同 LLM（如 OpenAI、Hugging Face、Cohere 等），并提供提示模板和输出解析器。

- 检索（Retrieval）：支持文档加载、文本分割、向量存储（如 Pinecone、Chroma）和检索器，便于实现 RAG 架构。

- 链（Chains）：将多个组件组合成一个端到端的任务管道，例如“检索文档 → 格式化提示 → 调用模型 → 输出结果”。

- 代理（Agents）：让 LLM 能够决定使用哪些工具（如搜索引擎、计算器、数据库查询）来完成任务，实现动态决策。

- 记忆（Memory）：在对话或多次调用中保持状态，例如存储历史消息以便模型理解上下文。

- 回调（Callbacks）：用于监控、日志记录和流式输出等。

#### 典型应用场景

文档问答：基于私有文档（PDF、网页等）构建问答系统。

智能对话机器人：具备长期记忆和工具调用能力的助手。

数据分析：通过自然语言查询数据库或电子表格。

代码理解与生成：分析代码库或自动编写代码。

### ReAct

ReAct 是一种用于构建智能代理（Agent）的提示工程框架，其名称源自 Reasoning（推理）和 Acting（行动）的结合。

#### 核心思想

传统的大语言模型通常只能基于内部知识生成文本，而无法与外部世界交互。ReAct 则通过提示引导模型：

推理（Reasoning）：模型输出对当前问题的思考过程，分析下一步需要做什么。

行动（Acting）：模型调用外部工具（如搜索引擎、计算器、数据库）或执行具体操作，获取新信息。

观察（Observation）：模型接收行动的结果，并将其作为下一步推理的依据。

通过这种“思考-行动-观察”的循环，模型能够动态调整策略，逐步逼近正确答案。

#### 工作流程示例

假设一个任务：“请找出电影《盗梦空间》的导演还执导过哪些科幻电影？”
ReAct 代理可能这样运作：

推理：我需要知道《盗梦空间》的导演是谁。

行动：调用一个知识检索工具，搜索“盗梦空间 导演”。

观察：得到结果“克里斯托弗·诺兰”。

推理：现在需要查找诺兰执导的科幻电影。

行动：搜索“克里斯托弗·诺兰 科幻电影 列表”。

观察：返回《星际穿越》《信条》《致命魔术》等。

最终回答：整合信息输出答案。

## 稠密检索

稠密检索是语义搜索的第一大核心类型，通过文本嵌入的相似度实现精准的结果筛选

## 重排器

重排器是语义搜索的第二大核心类型，重排器能够接收搜索查询与初始结果集，并根据相关性进行重新排序，从而显著提升结果质量

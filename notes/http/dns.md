# DNS 解析

> DNS: Domain Name System
> 可参考书籍[计算机网络（原书第 7 版）](https://book.douban.com/subject/30280001/)

## DNS 服务器类型分类

-   根 DNS 服务器：其作用是管理它的下一级，即 TLD DNS 服务器。
    -   <code>com</code>、<code>cn</code>、<code>org</code>与<code>edu</code> 等为顶级域名
    -   通过询问根 DNS 服务器，可以知道一个主机名对应的 TDL DNS 服务器的 IP 是多少，从而继续向 TDL DNS 服务器发起查询请求
-   顶级域（Top-level Domain，TLD） DNS 服务器：其作用提供了它的下一级，即权威 DNS 服务器的 IP 地址
-   权威 DNS 服务器：其作用是返回 主机-IP 的最终映射

## QA

### TLD 一定知道权威 DNS 服务器的 IP 么？

-   不一定。有时 TLD 只是知道中间的某个 DNS 服务器，再由这个中间 DNS 服务器去找到权威 DNS 服务器。

## DNS 记录：存储在权威服务器上的映射规则，常见类型：

-   A 记录：域名 → IPv4 地址。
-   AAAA 记录：域名 → IPv6 地址。
-   CNAME 记录：域名别名 → 另一个域名（如 www.example.com → example.com）。
-   MX 记录：指定接收域名的邮件服务器。
-   TXT 记录：存储文本信息（如验证、SPF 配置）。

TTL：记录的“存活时间”，决定结果可被缓存多久（单位：秒）。

## IPv4 与 IPv6

> 核心区别在于地址结构、容量、设计理念以及对现代网络需求的适应性

### 典型场景对比

IPv4 环境：
家庭路由器使用 NAT，手机和电脑共享一个公网 IPv4，外网无法直接访问内网设备。

IPv6 环境：
每个设备拥有公网 IPv6，可直接远程控制智能家居（如摄像头、灯光），无需端口映射。

为什么需要 IPv6？
地址枯竭：IPv4 地址已全部分配完毕，依赖 NAT 和私有地址导致网络复杂化。
物联网（IoT）：智能设备激增需海量独立 IP（冰箱、传感器等均可直连）。
性能优化：简化包头、取消 NAT 降低延迟（对游戏/视频会议至关重要）。
安全性：强制 IPSec 提供端到端加密（防监听、篡改）。
未来扩展：为元宇宙、量子网络等预留地址空间。
